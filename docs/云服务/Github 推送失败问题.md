# GitHub 上传代码 Timed out 问题解决（代理配置）

## 一、问题背景

在使用 Git 将本地代码上传到 GitHub 时，可能会出现：

- `Timed out`
- 无法连接 GitHub
- push 失败

👉 常见原因：**本地 Git 代理配置错误或未正确设置**

------

## 二、问题本质

Git 在访问 GitHub 时依赖网络代理：

- 配置了错误代理 → 无法连接
- 没有代理（在某些网络环境）→ 访问失败

------

## 三、解决思路

核心流程：

1. **检查是否设置代理**
2. **删除错误代理**
3. **重新设置正确代理**

------

## 四、具体操作步骤

### 1️⃣ 查看当前代理配置

```bash
git config --global http.proxy
git config --global https.proxy
```

📌 说明：

- 如果有输出 → 说明当前配置了代理
- 如果没有 → 表示未配置

------

### 2️⃣ 取消已有代理（关键步骤）

```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```

📌 作用：

- 清除错误或无效代理配置

------

### 3️⃣ 获取代理地址

一般代理格式：

```
127.0.0.1:端口号
```

Windows获取建议：

```text
网络 -> 手动设计代理 -> 地址+端口
```

📌 注意：

- 端口号必须和你本地代理工具一致（如 Clash、V2Ray 等）
- ❗不要照抄网上的端口（如 1080），必须用自己的

------

### 4️⃣ 重新设置代理

```bash
git config --global https.proxy 127.0.0.1:端口号
git config --global http.proxy 127.0.0.1:端口号
```

------

### 5️⃣ 再次尝试 push

```bash
git push -u origin master
```

------

## 五、核心总结

### ✅ 一句话本质

> Git push 超时 ≈ 代理配置错误

------

### ✅ 标准解决流程

```text
查看代理 → 删除代理 → 设置正确代理 → 重新 push
```

------

### ✅ 常见坑

- ❌ 使用错误端口（最常见）
- ❌ 开了代理软件但端口不一致
- ❌ 忘记取消旧代理
- ❌ 同时存在多个代理配置

------

## 六、补充说明

- Git 代理是 **全局配置（--global）**
- 影响所有 Git 仓库
- 修改后立即生效，无需重启

------

## 七、扩展（可选）

### 查看所有 Git 配置

```bash
git config --list
```

------

## 八、适用场景

- GitHub push 超时
- clone 失败
- 公司/校园网络访问 GitHub 异常

------
