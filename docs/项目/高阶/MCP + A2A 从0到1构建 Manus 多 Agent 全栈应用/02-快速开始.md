# 第2章：快速开始

## 一、环境准备

### 1.1 系统要求

| 要求 | 最低配置 | 推荐配置 |
|-----|---------|---------|
| 操作系统 | Windows 10 / macOS 10.15 / Ubuntu 18.04 | Windows 11 / macOS 12 / Ubuntu 22.04 |
| CPU | 4 核 | 8 核以上 |
| 内存 | 8 GB | 16 GB 以上 |
| 磁盘 | 20 GB | 50 GB 以上 |
| Docker | 20.10+ | 24.0+ |
| Docker Compose | 2.0+ | 2.20+ |

### 1.2 安装 Docker

#### Windows

1. 下载 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop)
2. 运行安装程序，按提示完成安装
3. 重启电脑
4. 打开 Docker Desktop，确保 Docker 正在运行

```powershell
# 验证安装
docker --version
docker compose version
```

#### macOS

1. 下载 [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop)
2. 将 Docker 拖入 Applications 文件夹
3. 打开 Docker Desktop
4. 等待 Docker 启动完成

```bash
# 验证安装
docker --version
docker compose version
```

#### Linux (Ubuntu)

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 添加当前用户到 docker 组
sudo usermod -aG docker $USER

# 安装 Docker Compose
sudo apt install docker-compose-plugin

# 验证安装
docker --version
docker compose version
```

### 1.3 获取项目代码

```bash
# 克隆项目
git clone https://github.com/your-repo/mooc-manus.git

# 进入项目目录
cd mooc-manus
```

## 二、配置说明

### 2.1 环境变量配置

创建 `.env` 文件：

```bash
# 复制示例文件
cp .env.example .env
```

编辑 `.env` 文件：

```bash
# ============================================
# 必须修改的配置
# ============================================

# 腾讯云 COS 配置（文件存储）
COS_SECRET_ID=your_cos_secret_id_here
COS_SECRET_KEY=your_cos_secret_key_here
COS_BUCKET=your_cos_bucket_here
COS_REGION=ap-shanghai

# ============================================
# 可选修改的配置
# ============================================

# 数据库配置
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_DB=manus

# Redis 配置
REDIS_HOST=redis
REDIS_PORT=6379

# 服务端口
NGINX_PORT=8088

# 沙箱配置
SANDBOX_IMAGE=manus-sandbox:latest
```

### 2.2 AI 模型配置

编辑 `api/config.yaml`：

```yaml
# LLM 配置
llm_config:
  base_url: https://api.deepseek.com/  # 或其他 OpenAI 兼容 API
  api_key: your_api_key_here           # 你的 API Key
  model_name: deepseek-chat            # 模型名称

# Agent 配置
agent_config:
  max_iterations: 10      # 最大迭代次数
  max_retries: 3          # 最大重试次数

# MCP 服务器配置（可选）
mcpServers:
  mcp_example:
    enabled: false
    transport: stdio
    command: python
    args:
      - "-m"
      - "mcp_server_example"

# A2A 服务器配置（可选）
a2a_servers:
  - id: example_agent
    enabled: false
    base_url: http://localhost:8001
```

### 2.3 支持的 AI 模型

| 模型提供商 | base_url | model_name |
|-----------|----------|------------|
| DeepSeek | https://api.deepseek.com/ | deepseek-chat / deepseek-reasoner |
| OpenAI | https://api.openai.com/v1 | gpt-4 / gpt-4o / gpt-3.5-turbo |
| Claude | https://api.anthropic.com/v1 | claude-3-opus / claude-3-sonnet |
| 本地模型 | http://localhost:11434/v1 | llama2 / mistral |

## 三、启动服务

### 3.1 一键启动

```bash
# 构建并启动所有服务
docker compose up -d --build
```

### 3.2 查看服务状态

```bash
# 查看所有容器状态
docker compose ps

# 预期输出
NAME              STATUS    PORTS
manus-nginx       running   0.0.0.0:8088->80/tcp
manus-ui          running   3000/tcp
manus-api         running   8000/tcp
manus-postgres    running   5432/tcp
manus-redis       running   6379/tcp
manus-sandbox     running   6080/tcp, 5900/tcp
```

### 3.3 查看日志

```bash
# 查看所有服务日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f manus-api
docker compose logs -f manus-ui
docker compose logs -f manus-sandbox
```

### 3.4 容器说明

| 容器名称 | 服务 | 说明 |
|---------|------|------|
| manus-nginx | Nginx | 反向代理网关，唯一对外暴露端口 |
| manus-ui | Next.js | 前端 UI 服务 |
| manus-api | FastAPI | 后端 API 服务 |
| manus-postgres | PostgreSQL | 数据库 |
| manus-redis | Redis | 缓存和消息队列 |
| manus-sandbox | Sandbox | 沙箱环境（Chrome + VNC） |

## 四、访问系统

### 4.1 访问地址

```
主界面：http://localhost:8088
API 文档：http://localhost:8088/api/docs
VNC 预览：http://localhost:8088 (在界面中查看)
```

### 4.2 首次使用

1. **打开浏览器**，访问 `http://localhost:8088`

2. **创建会话**：点击左侧 "新建会话" 按钮

3. **发送消息**：在输入框中输入任务，例如：
   ```
   帮我搜索 Python 教程并整理成文档
   ```

4. **观察执行过程**：
   - 右侧显示 Agent 的思考过程
   - 中间显示 VNC 实时预览
   - 底部显示文件列表

## 五、本地开发

### 5.1 后端开发

```bash
# 进入 api 目录
cd api

# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate

# 安装依赖
pip install uv
uv pip install -r requirements.txt

# 安装 Playwright 浏览器
playwright install

# 启动开发服务器
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 5.2 前端开发

```bash
# 进入 ui 目录
cd ui

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 访问 http://localhost:3000
```

### 5.3 数据库迁移

```bash
# 进入 api 目录
cd api

# 生成迁移脚本
alembic revision --autogenerate -m "描述"

# 执行迁移
alembic upgrade head

# 回滚
alembic downgrade -1
```

## 六、常用命令

### 6.1 Docker 命令

```bash
# 启动所有服务
docker compose up -d

# 停止所有服务
docker compose down

# 重启单个服务
docker compose restart manus-api

# 重新构建并启动
docker compose up -d --build

# 查看日志
docker compose logs -f manus-api

# 进入容器
docker exec -it manus-api bash

# 停止并清除数据卷（谨慎操作）
docker compose down -v
```

### 6.2 开发命令

```bash
# 后端
cd api
uvicorn app.main:app --reload    # 启动开发服务器
pytest                            # 运行测试
alembic upgrade head              # 数据库迁移

# 前端
cd ui
npm run dev                       # 启动开发服务器
npm run build                     # 构建生产版本
npm run lint                      # 代码检查
```

## 七、常见问题

### 7.1 端口冲突

**问题**：8088 端口被占用

**解决**：修改 `.env` 文件中的 `NGINX_PORT`

```bash
# 修改端口
NGINX_PORT=8089
```

### 7.2 Docker 内存不足

**问题**：容器启动失败，提示内存不足

**解决**：增加 Docker 内存限制

1. 打开 Docker Desktop
2. 进入 Settings -> Resources
3. 增加 Memory 限制到 8GB 以上

### 7.3 API Key 无效

**问题**：调用 AI 模型失败

**解决**：检查 `api/config.yaml` 中的 API Key 配置

```yaml
llm_config:
  api_key: sk-xxxxx  # 确保正确
```

### 7.4 数据库连接失败

**问题**：API 启动失败，提示数据库连接错误

**解决**：

```bash
# 检查数据库容器状态
docker compose ps manus-postgres

# 查看数据库日志
docker compose logs manus-postgres

# 重启数据库
docker compose restart manus-postgres
```

### 7.5 VNC 无法连接

**问题**：无法看到浏览器预览

**解决**：

```bash
# 检查沙箱容器状态
docker compose ps manus-sandbox

# 重启沙箱
docker compose restart manus-sandbox
```
