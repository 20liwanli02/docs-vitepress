## 三、面试常见问题

### 3.1 项目相关

**Q1: 请介绍一下这个项目？**

> 这是一个通用 AI Agent 系统，采用 Planner + ReAct 双 Agent 协作架构。PlannerAgent 负责任务规划和分解，ReActAgent 负责任务执行和工具调用。系统支持浏览器自动化、Shell 命令执行、文件操作等多种工具，并集成了 MCP 和 A2A 协议，可以接入外部工具生态和远程 Agent。

**Q2: 为什么选择双 Agent 架构？**

> 职责分离是主要原因。规划需要全局视角和抽象思维，执行需要具体操作和工具调用。分离后：
> 1. 各司其职，更专注
> 2. 可以使用不同的 Prompt 优化各自能力
> 3. 便于调试和维护
> 4. 可以独立扩展

**Q3: ReAct 模式是什么？**

> ReAct = Reasoning + Acting。是一种让 Agent 交替进行思考和行动的模式：
> 1. Thought: 思考下一步该做什么
> 2. Action: 选择并调用工具
> 3. Observation: 观察工具返回结果
> 4. 循环直到任务完成
> 
> 这种模式让 Agent 能够处理复杂的多步骤任务。

**Q4: 如何处理 Agent 执行失败？**

> 多层容错机制：
> 1. LLM 调用重试：失败后自动重试
> 2. 工具调用重试：工具执行失败后重试
> 3. 计划动态调整：PlannerAgent 根据失败结果调整计划
> 4. 优雅降级：将错误信息返回给 LLM，让它决定下一步

**Q5: 如何保证沙箱安全？**

> 1. Docker 容器隔离：每个会话独立容器
> 2. 资源限制：CPU、内存、磁盘限制
> 3. 网络隔离：可选的网络隔离
> 4. 命令白名单：限制可执行的命令
> 5. 文件系统隔离：限制可访问的目录

### 3.2 技术相关

**Q6: 为什么选择 FastAPI 而不是 Flask/Django？**

> 1. 原生异步支持：Agent 任务可能运行很长时间，异步是必须的
> 2. 自动 API 文档：开发效率高
> 3. Pydantic 集成：数据验证方便
> 4. 性能优秀：适合高并发场景

**Q7: SSE 和 WebSocket 的区别？**

> SSE (Server-Sent Events):
> - 单向通信（服务器到客户端）
> - 基于 HTTP
> - 自动重连
> - 适合服务器推送场景
> 
> WebSocket:
> - 双向通信
> - 独立协议
> - 需要手动重连
> - 适合实时交互场景
> 
> 本项目用 SSE 推送 Agent 执行状态，用 WebSocket 实现 VNC 连接。

**Q8: MCP 和 A2A 的区别？**

> MCP (Model Context Protocol):
> - Agent 与工具的通信协议
> - 工具是被动的，等待调用
> - 适合集成外部工具
> 
> A2A (Agent-to-Agent):
> - Agent 之间的通信协议
> - 远程 Agent 是自主的
> - 适合多 Agent 协作
