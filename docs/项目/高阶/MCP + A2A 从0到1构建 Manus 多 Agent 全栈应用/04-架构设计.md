### 2.2 各层职责

| 层级 | 职责 | 示例 |
|-----|------|------|
| **接口层** | 接收请求、参数验证、返回响应 | `session_routes.py` |
| **应用层** | 业务编排、事务管理 | `AgentService` |
| **领域层** | 核心业务逻辑、领域模型 | `BaseAgent`、`Plan` |
| **基础设施层** | 技术实现、外部服务集成 | `OpenAILLM`、`PostgreSQL` |

### 2.3 依赖规则

```
依赖方向：从外向内

接口层 ──────► 应用层 ──────► 领域层
    │             │             │
    │             │             │
    └─────────────┴─────────────┘
                  │
                  ▼
            基础设施层

规则：
1. 内层不依赖外层
2. 领域层不依赖任何外层
3. 基础设施层实现领域层定义的接口
```

## 七、设计原则

### 7.1 SOLID 原则应用

| 原则 | 应用 |
|-----|------|
| **S** 单一职责 | 每个 Agent 只负责一类任务 |
| **O** 开闭原则 | 通过继承 BaseTool 扩展新工具 |
| **L** 里氏替换 | 所有 Agent 可替换 BaseAgent |
| **I** 接口隔离 | 定义最小化接口（Protocol） |
| **D** 依赖倒置 | 领域层定义接口，基础设施层实现 |

### 7.2 设计模式应用

| 模式 | 应用场景 |
|-----|---------|
| **工厂模式** | 创建 Agent、Tool 实例 |
| **策略模式** | 不同工具的调用策略 |
| **状态模式** | Flow 状态机 |
| **观察者模式** | 事件驱动架构 |
| **装饰器模式** | @tool 装饰器 |
