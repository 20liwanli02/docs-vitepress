### 📋 核心概念：Stripe 是什么？

**Stripe** 是一个第三方支付平台，类似于支付宝、微信支付，但主要面向国际支付。

| 概念                 | 解释                                          |
| -------------------- | --------------------------------------------- |
| **Checkout Session** | 一次支付会话，包含商品信息、金额、支付页面URL |
| **PaymentIntent**    | 支付意向，Stripe 内部追踪每一笔支付的核心对象 |
| **Webhook**          | Stripe 主动回调你服务器的机制，通知支付结果   |

### 🔄 完整流程：用户如何成为 VIP？

#### 第一步：用户点击"开通 VIP"

前端调用：
```java

```

#### 第二步：后端创建 Stripe Checkout Session

[PaymentServiceImpl.createVipPaymentSession()](file:///d:/TPROJECT/code/ai-passage-creator-main/passage-backend/src/main/java/com/yupi/template/service/impl/PaymentServiceImpl.java#L36-L44)：

```java
public String createVipPaymentSession(Long userId) {
    // 1. 校验用户不是 VIP
    validateNotVip(user);
    
    // 2. 创建 Stripe Session
    Session session = createStripeSession(userId, ProductTypeEnum.VIP_PERMANENT);
    //   这个 Session 包含:
    //   - 商品: 永久会员 ($199)
    //   - metadata: userId, productType
    //   - successUrl: 支付成功跳转页面
    //   - cancelUrl: 支付取消跳转页面
    
    // 3. 保存支付记录（状态 = PENDING）
    savePaymentRecord(userId, session, productType);
    
    // 4. 返回 Stripe 支付页面的 URL
    return session.getUrl();
}
```

**Stripe Checkout Session** 是什么？它是 Stripe 提供的一种**托管支付页面**，你不需要自己写支付界面，Stripe 会给你一个 URL，用户跳转到那个页面完成支付。

#### 第四步：用户完成支付，Stripe 回调我们的服务器

**关键问题**：Stripe 怎么知道支付成功了？

Stripe 会 POST 到我们配置的 Webhook URL：`/webhook/stripe`

[StripeWebhookController](file:///d:/TPROJECT/code/ai-passage-creator-main/passage-backend/src/main/java/com/yupi/template/controller/StripeWebhookController.java)：

```java
@PostMapping("/webhook/stripe")
public String handleStripeWebhook(@RequestBody String payload,
                                   @RequestHeader("Stripe-Signature") String sigHeader) {
    
    // 1. 验证请求确实来自 Stripe（防伪造）
    Event event = paymentService.constructEvent(payload, sigHeader);
    
    // 2. 判断事件类型
    if ("checkout.session.completed".equals(event.getType())) {
        // 支付成功！
        Session session = ...;
        paymentService.handlePaymentSuccess(session);  // 处理支付成功逻辑
    }
    
    return "success";  // 返回给 Stripe
}
```

#### 第五步：处理支付成功

[PaymentServiceImpl.handlePaymentSuccess()](file:///d:/TPROJECT/code/ai-passage-creator-main/passage-backend/src/main/java/com/yupi/template/service/impl/PaymentServiceImpl.java#L48-L70)：

```java
@Transactional  // 保证事务
public void handlePaymentSuccess(Session session) {
    String sessionId = session.getId();
    String userId = session.getMetadata().get("userId");  // 从 metadata 获取 userId
    
    // 1. 幂等性检查：防止重复处理
    PaymentRecord record = findPaymentRecordBySessionId(sessionId);
    if (PaymentStatusEnum.SUCCEEDED.equals(record.getStatus())) {
        return;  // 已经处理过了，直接返回
    }
    
    // 2. 更新支付记录状态 = SUCCEEDED
    updatePaymentStatus(record.getId(), PaymentStatusEnum.SUCCEEDED, paymentIntentId);
    
    // 3. 升级用户为 VIP
    upgradeUserToVip(Long.valueOf(userId));
}
```

**升级用户为 VIP** 的具体操作：
```java
private void upgradeUserToVip(Long userId) {
    User user = new User();
    user.setId(userId);
    user.setVipTime(LocalDateTime.now());      // 设置 VIP 开通时间
    user.setUserRole("vip");                    // 改角色为 vip
    userMapper.update(user);
}
```

### 🔙 第六步：退款流程

用户想取消 VIP？可以申请退款：

```java
POST /payment/refund?reason=不想要了
```

```java
public boolean handleRefund(Long userId, String reason) {
    // 1. 校验用户是 VIP
    validateIsVip(user);
    
    // 2. 找到最近的支付记录
    PaymentRecord paymentRecord = findLatestSuccessfulPayment(userId);
    
    // 3. 调用 Stripe API 发起退款
    Refund refund = Stripe.Refund.create(params);
    
    // 4. 更新支付记录状态 = REFUNDED
    updateRefundRecord(paymentRecord.getId(), reason);
    
    // 5. 撤销用户 VIP 身份
    revokeVipStatus(userId);  // vipTime = null, role = "user", quota = 5
}
```

### 📊 数据库记录

#### PaymentRecord（支付记录表）

| 字段                    | 说明                              |
| ----------------------- | --------------------------------- |
| `stripeSessionId`       | Stripe Session ID（用于查找记录） |
| `stripePaymentIntentId` | Stripe 支付意向 ID（用于退款）    |
| `amount`                | 金额 ($199)                       |
| `status`                | PENDING → SUCCEEDED / REFUNDED    |
| `productType`           | VIP_PERMANENT                     |

#### User（用户表）关键字段

| 字段       | 普通用户 | VIP 用户 |
| ---------- | -------- | -------- |
| `userRole` | "user"   | "vip"    |
| `vipTime`  | null     | 开通时间 |
| `quota`    | 5        | 不限     |

### 🔐 安全性设计

#### 1. Webhook 签名验证

Stripe 回调时携带 `Stripe-Signature` 头，后端验证：

```java
Event event = Webhook.constructEvent(payload, sigHeader, webhookSecret);
```

如果签名不匹配，说明请求不是 Stripe 发的，是伪造的。

#### 2. 幂等性检查

```java
// 防止 Stripe 重复回调导致重复处理
if (PaymentStatusEnum.SUCCEEDED.equals(record.getStatus())) {
    return;  // 已经处理过了
}
```

#### 3. 用户身份校验

```java
// 创建支付前：不能已经是 VIP
validateNotVip(user);

// 退款前：必须是 VIP
validateIsVip(user);
```

### 🔑 为什么需要 Webhook？

**核心问题**：支付完成后，Stripe 怎么通知我们"支付成功了"？

| 方案                                     | 问题                        |
| ---------------------------------------- | --------------------------- |
| 前端回调（Stripe 返回后跳转 successUrl） | 不安全，用户可以伪造        |
| 前端轮询查状态                           | 慢、浪费资源                |
| **Stripe Webhook（后端回调）**           | ✅ 安全可靠，Stripe 官方推荐 |

Webhook 就是 Stripe 主动 POST 到我们服务器告诉我们支付结果，这才是真正的"支付完成"信号。
