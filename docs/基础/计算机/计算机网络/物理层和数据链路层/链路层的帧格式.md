# 以太网帧格式与 MAC 地址工作机制笔记

## 一、数据链路层的封装回顾

当数据从上层传到数据链路层时：

```text
应用层 → 运输层 → 网络层 → 数据链路层
```

👉 此时数据链路层的**数据载荷**包含：

- 应用层数据
- 应用层首部
- 运输层首部
- 网络层首部

------

## 二、数据链路层的核心操作：封装成帧

👉 数据链路层会对数据进行：

```text
加首部 + 加尾部 → 封装成帧（Frame）
```

------

## 三、以太网 II 帧格式（重点）

### 帧结构如下：

```text
| 目标MAC(6B) | 源MAC(6B) | 类型(2B) | 数据载荷(46~1500B) | FCS(4B) |
```

------

## 四、各字段详解

### 1️⃣ 目标 MAC 地址（6 字节）

- 表示：**发给谁**
- 接收端用于判断：
  - 是否接收该帧

------

### 2️⃣ 源 MAC 地址（6 字节）

- 表示：**从哪来**
- 用途：
  - 接收方可以回复数据

------

### 3️⃣ 类型字段（2 字节）

👉 作用：

> 标识数据载荷使用的上层协议

------

### 常见值：

- `0x0800` → IPv4
- `0x0806` → ARP
- `0x86DD` → IPv6

------

### 4️⃣ 数据载荷（Payload）

👉 内容：

- 来自网络层的数据（如 IP 数据包）

------

👉 长度限制：

- 最小：46 字节
- 最大：1500 字节

------

### ❗为什么有最大限制？

👉 如果超过 1500 字节：

- 会在上层（网络层）进行**分片（Fragmentation）**

------

### 5️⃣ FCS（帧校验序列，4 字节）

👉 作用：

> 检测数据是否在传输过程中发生错误

------

👉 原理：

- 基于校验算法（如 CRC）
- 检测比特是否发生“跳变”

------

👉 注意：

- ❗抓包中**看不到 FCS**
- 原因：
  - 网卡已完成校验并移除

------

## 五、抓包验证（Wireshark）

在抓包工具中可以看到：

### 示例结构：

- Ethernet II
  - Destination（目标 MAC）
  - Source（源 MAC）
  - Type（协议类型）

------

### 说明：

- 左边：解析后的字段
- 右边：真实二进制数据（十六进制表示）

------

## 六、接收端如何解析帧（重点）

👉 核心依据：

> **协议规定的格式**

------

### 解析步骤：

1. 读取前 6 字节 → 目标 MAC
2. 读取后 6 字节 → 源 MAC
3. 读取 2 字节 → 类型
4. 剩余部分 → 数据载荷

------

👉 本质：

```text
按照“固定结构”切割二进制数据
```

------

## 七、MAC 地址如何起作用

### 接收端判断逻辑：

```text
目标MAC == 本机MAC ?
```

------

### 情况：

- ✔ 相同 → 接收
- ❌ 不同 → 丢弃
- ✔ 广播（FF:FF:FF:FF:FF:FF）→ 全部接收

------

## 八、协议的重要性（核心理解）

👉 没有协议：

- 0101 无意义（无法解析）

👉 有协议：

- 可以明确：
  - 哪些字节表示什么
  - 如何拆分数据

------

## 九、数据逐层解析过程

```text
链路层解析 → 网络层解析 → 运输层解析 → 应用层解析
```

------

👉 每一层：

- 读取自己的首部
- 提取信息
- 去掉首部
- 上交数据

------

## 十、关键细节总结

### 1️⃣ 帧头长度固定

- 以太网 II：
  - 14 字节

------

### 2️⃣ 分隔符不重要

- Windows：`-`
- Linux / Wireshark：`:`

👉 本质都是：

```text
十六进制表示
```

------

### 3️⃣ 数据本质不变

- 一直都是：

```text
0101（二进制）
```

------

## 十一、核心总结

### 一句话总结帧结构

> **数据链路层通过固定格式（帧）组织数据，并通过 MAC 地址实现局域网内通信**

------

### 核心流程

```text
封装（加头尾） → 发送 → 接收 → 按协议解析
```

------

### 最关键理解

> **协议让“无序的二进制数据”变成“有结构的信息”**

------

如果你继续深入学习，下一步最关键的是：

👉 **ARP 协议（IP → MAC 的映射）**

这个是把“网络层”和“链路层”真正打通的关键机制。
