# 数据链路层协议与总结笔记

## 一、数据链路层回顾

### 核心作用

> **解决数据包在一个网络 / 一段链路上的传输问题**

------

### 链路的定义

- 一段链路 = **两个节点之间的物理连接**
- 条件：
  - 中间**没有路由器**

------

### 举例理解

- ✔ 同一链路（局域网）：
  - 主机 ↔ 主机
  - 主机 ↔ 路由器接口
- ❌ 不同链路：
  - 跨路由器通信

------

## 二、MAC 地址机制（核心）

### 基本特性

- 长度：6 字节（48 位）
- 又叫：物理地址
- 存储位置：网卡（NIC）
- 特点：
  - 出厂固化（全球唯一）
  - 操作系统可“伪造”（可修改）

------

### 特殊 MAC 地址

- 广播地址：

```text
FF:FF:FF:FF:FF:FF
```

👉 含义：

- 发给**当前网络内所有主机**

------

### 工作机制

#### 发送端：

- 在帧头中加入：
  - 源 MAC
  - 目的 MAC

------

#### 接收端（网卡处理）：

```text
目的MAC == 本机MAC → 接收  
目的MAC != 本机MAC → 丢弃  
目的MAC == 广播地址 → 接收
```

------

## 三、以太网 II 帧格式（回顾）

```text
| 目标MAC(6B) | 源MAC(6B) | 类型(2B) | 数据(46~1500B) | FCS(4B) |
```

------

### 关键点

- 帧头：14 字节
- 数据范围：46 ~ 1500 字节
- FCS：
  - 用于差错检测
  - ❗抓包中看不到（网卡已处理）

------

### 类型字段作用

👉 标识上层协议：

- IPv4（0x0800）
- ARP（0x0806）
- IPv6（0x86DD）

------

## 四、数据过大如何处理

👉 超过 1500 字节：

- 上层（网络层）进行：
  - **分片（Fragmentation）**

------

## 五、数据链路层核心协议

------

## 1️⃣ CSMA/CD（碰撞检测）

### 全称

```text
Carrier Sense Multiple Access with Collision Detection
```

------

### 适用场景

👉 **有线局域网（共享总线以太网）**

------

### 核心问题

> 多台主机共享一条通信介质 → 容易发生碰撞

------

### 工作机制

1. 发送前监听信道（载波监听）
2. 空闲才发送
3. 发送过程中检测碰撞
4. 若发生碰撞：
   - 停止发送
   - 随机等待
   - 重新发送

------

### 本质

> **“先发 + 发现冲突再处理”**

------

## 2️⃣ CSMA/CA（碰撞避免）

### 全称

```text
Carrier Sense Multiple Access with Collision Avoidance
```

------

### 适用场景

👉 **无线局域网（WiFi）**

------

### 为什么不用 CD？

- 无线环境：
  - 难以检测碰撞
  - 硬件成本高
  - 检测不准确

------

### 核心思想

> **尽量避免碰撞，而不是检测碰撞**

------

### 特点

- 通过等待、控制发送时机降低冲突概率
- ❗不能完全避免碰撞

------

### 对比总结

| 协议    | 场景 | 思想            |
| ------- | ---- | --------------- |
| CSMA/CD | 有线 | 先发 → 检测冲突 |
| CSMA/CA | 无线 | 先避免 → 再发   |

------

## 3️⃣ PPP（点对点协议）

### 全称

```text
Point-to-Point Protocol
```

------

### 核心特点

> **用于两点之间直接通信**

------

### 应用场景

#### 1️⃣ 主机 ↔ ISP（运营商）

- 常见形式：

```text
PPPoE（PPP over Ethernet）
```

👉 示例：

- 家庭宽带拨号
- 路由器上网方式

------

#### 2️⃣ 路由器 ↔ 路由器

- 点对点链路通信

------

## 六、实践补充（路由器访问）

### 查找路由器 IP

Windows：

```bash
ipconfig /all
```

👉 关键字段：

- 默认网关 = 路由器 IP

------

### 访问方式

浏览器输入：

```text
http://默认网关地址
```

------

## 七、整体流程总结

### 数据链路层工作流程

```text
封装帧 → 添加MAC地址 → 发送 → 接收 → 校验 → 解封装
```

------

## 八、本节核心总结

### 1️⃣ 数据链路层解决的问题

> **同一链路（局域网）内如何可靠传输数据**

------

### 2️⃣ 三大核心机制

- MAC 地址 → 标识设备
- 帧结构 → 组织数据
- 协议（CSMA/CD、CA、PPP） → 控制通信

------

### 3️⃣ 最关键理解

> **数据链路层 = 局域网通信规则制定者**

------

### 4️⃣ 一句话总结

> **链路层通过 MAC 地址 + 帧格式 + 协议，实现局域网内高效可靠通信**

------
