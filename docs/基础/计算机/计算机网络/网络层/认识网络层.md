# 🌐 网络层（Network Layer）

## 一、网络层在体系结构中的位置

### 📊 五层模型位置

| 层级       | 名称        | 作用       |
| ---------- | ----------- | ---------- |
| 应用层     | Application | 用户应用   |
| 运输层     | Transport   | 端到端通信 |
| ⭐ 网络层   | Network     | 跨网络传输 |
| 数据链路层 | Data Link   | 局域网通信 |
| 物理层     | Physical    | 比特传输   |

------

### 📌 关键定位

```text
网络层 = 数据链路层之上 + 运输层之下
```

------

## 二、网络层解决的核心问题

------

### 📌 各层对比（非常重要）

| 层         | 解决问题          | 范围     |
| ---------- | ----------------- | -------- |
| 物理层     | 01 ↔ 电信号       | 单条链路 |
| 数据链路层 | 帧传输            | 局域网   |
| ⭐ 网络层   | 跨网络传输 + 路由 | 广域网   |

------

### 📌 本质总结

```text
网络层 = 让数据包可以“跨网络”传输
```

------

## 三、网络层核心协议

------

### 📊 主要协议

| 协议 | 作用            |
| ---- | --------------- |
| IP   | 地址 + 路由     |
| ARP  | IP → MAC        |
| ICMP | 差错控制 / ping |

------

### 📌 快速理解

```text
IP   → 定位（去哪）
ARP  → 找人（找MAC）
ICMP → 检测（通不通）
```

------

## 四、网络层核心场景（跨网络通信）

------

### 📌 场景说明

```text
主机A（网络1） → 主机B（网络2）
```

------

### 📊 网络结构

```
局域网1 ── 路由器 ── 局域网2 ── 路由器 ── 局域网3
```

------

### 📌 关键点

- 不再是“一个网内通信”
- 而是：

```text
跨多个网络传输
```

------

## 五、路由（Routing）核心概念

------

### 📌 什么是路由？

```text
路由 = 路径选择
```

------

### 📊 举例

从 A 到 B：

```
路径1：A → R1 → R2 → B
路径2：A → R3 → R4 → B
```

------

### 📌 关键问题

```text
到底走哪条路径？
```

👉 由 **路由器** 决定

------

## 六、路由器（核心设备）

------

### 📌 定位

```text
路由器 = 网络层设备（L3设备）
```

------

### 📊 能力

| 能力     | 说明             |
| -------- | ---------------- |
| 解析 IP  | 查看目标地址     |
| 查路由表 | 决定下一跳       |
| 转发数据 | 发送到下一个网络 |

------

## 七、数据包跨网络传输流程（重点）

------

## 🚀 总体流程

```text
发送方 → 路由器 → 路由器 → 接收方
```

------

### 1️⃣ 发送方（封装）

```text
应用层数据
 ↓
网络层（加IP头）
 ↓
链路层（加MAC头+尾）
 ↓
物理层（转为信号）
```

------

### 2️⃣ 路由器（关键）

#### 📌 步骤

```text
1. 接收信号
2. 逐层解封（到网络层）
3. 读取 IP 地址
4. 查路由表
5. 决定出口
6. 重新封装
7. 转发
```

------

### 📌 核心特点

```text
路由器 = 解封 → 判断 → 再封装 → 转发
```

------

### 3️⃣ 接收方（解封）

```text
物理层 → 链路层 → 网络层 → 上层
```

------

## 八、封装 & 解封（高频考点）

------

### 📌 发送端

```text
上 → 下（封装）
```

------

### 📌 接收端

```text
下 → 上（解封）
```

------

### 📊 总结

| 方向 | 动作 |
| ---- | ---- |
| 发送 | 封装 |
| 接收 | 解封 |

------

## 九、关键理解（非常重要）

------

### 📌 为什么需要网络层？

```text
链路层只能解决：
👉 一个局域网内通信

网络层解决：
👉 不同网络之间通信
```

------

### 📌 本质区别

| 层         | 能力     |
| ---------- | -------- |
| 数据链路层 | 点到点   |
| 网络层     | 多跳传输 |

------

## 十、面试高频问题总结

------

### Q1：网络层是干什么的？

```text
负责数据包在多个网络之间传输 + 路由选择
```

------

### Q2：路由是什么？

```text
路径选择过程
```

------

### Q3：路由器做了什么？

```text
解封 → 查表 → 转发 → 再封装
```

------

### Q4：网络层 vs 数据链路层？

| 对比 | 数据链路层 | 网络层 |
| ---- | ---------- | ------ |
| 范围 | 局域网     | 跨网络 |
| 地址 | MAC        | IP     |
| 设备 | 交换机     | 路由器 |

------

## 🔥 一句话总结（面试必杀）

```text
网络层负责通过 IP 地址实现跨网络通信，
并由路由器根据路由表决定数据包的转发路径。
```
