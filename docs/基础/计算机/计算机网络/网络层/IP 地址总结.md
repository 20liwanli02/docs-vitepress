# IPv4 地址体系总结

## 一、IPv4 地址基础

### 📌 基本结构

```text
IPv4 = 32 位（二进制）
```

划分为：

```text
网络号 + 主机号
```

------

### 📊 含义

| 部分   | 作用                     |
| ------ | ------------------------ |
| 网络号 | 标识属于哪个网络         |
| 主机号 | 标识该网络中的第几台主机 |

------

### 📌 核心规则

- 同一网络：
  - ✅ 网络号必须相同
  - ❌ 主机号不能重复

------

### 📌 类比理解

```text
网络号 = 小区
主机号 = 门牌号
```

------

## 二、IP 地址分类（ABC 类）

------

### 📊 分类规则（按前缀）

| 类别 | 前缀 | 网络号 | 主机号 |
| ---- | ---- | ------ | ------ |
| A 类 | 0    | 8 位   | 24 位  |
| B 类 | 10   | 16 位  | 16 位  |
| C 类 | 110  | 24 位  | 8 位   |

------

### 📊 特点

| 类别 | 网络数量 | 主机数量 |
| ---- | -------- | -------- |
| A 类 | 少       | 多       |
| B 类 | 中       | 中       |
| C 类 | 多       | 少       |

------

### ❗ 问题

```text
分类固定 → 不灵活 → IP 浪费
```

------

## 三、子网划分（Subnetting）

------

### 📌 核心思想

```text
从主机号借位 → 划分子网
```

------

### 📊 影响

| 项目       | 变化 |
| ---------- | ---- |
| 子网数量   | 增加 |
| 每网主机数 | 减少 |

------

### 📌 示例

```text
/24 → /25
```

| 项目       | 数值 |
| ---------- | ---- |
| 子网数     | 2    |
| 每网主机数 | 126  |

------

## 四、子网掩码（Subnet Mask）

------

### 📌 作用

```text
标识：网络位 vs 主机位
```

------

### 📌 表示规则

| 位   | 含义   |
| ---- | ------ |
| 1    | 网络位 |
| 0    | 主机位 |

------

### 📊 示例

```text
255.255.255.128 = /25
```

------

## 五、CIDR（重点）

------

### 📌 表示方式

```text
IP/网络位数
```

------

### 📊 示例

```text
192.168.1.1/24
```

------

### 📌 优势

| 优势     | 说明         |
| -------- | ------------ |
| 简洁     | 替代子网掩码 |
| 灵活     | 不依赖 ABC   |
| 现代标准 | 实际网络使用 |

------

### 📌 本质

```text
CIDR = 分类 + 子网划分 的统一
```

------

## 六、同一网络判断（核心难点）

------

### 📌 正确方法（面试重点）

```text
发送方：
1. 自己IP & 自己掩码 → 自己网络
2. 目标IP & 自己掩码 → 目标网络（视角）
3. 对比
```

------

### ❗ 关键点

```text
必须使用“发送方的子网掩码”
```

------

## 七、通信逻辑（本质）

------

```text
是否同网段？
    ↓
是 → ARP → 直接通信
否 → 发给网关（路由器）
```

------

## 八、为什么会通信失败？（经典问题）

------

### 📊 原因

| 问题     | 原因         |
| -------- | ------------ |
| 发不出去 | 不在同一网段 |
| 单向通信 | 掩码不同     |
| 无法回包 | 无网关       |
| ARP 失败 | 无法获取 MAC |

------

## 九、公网 IP vs 私网 IP

------

### 📊 对比

| 类型    | 用途   | 是否唯一 |
| ------- | ------ | -------- |
| 公网 IP | 互联网 | ✅ 唯一   |
| 私网 IP | 局域网 | ❌ 可重复 |

------

### 📌 私网地址范围

| 类别 | 范围           |
| ---- | -------------- |
| A    | 10.0.0.0/8     |
| B    | 172.16.0.0/12  |
| C    | 192.168.0.0/16 |

------

## 十、NAT（网络地址转换）

------

### 📌 作用

```text
私网 IP → 公网 IP
```

------

### 📊 工作流程

```text
内网主机 → NAT路由器 → 公网
公网返回 → NAT路由器 → 内网主机
```

------

### 📌 核心

```text
IP 映射表
```

------

## 十一、NAPT（端口复用，重点）

------

### 📌 为什么需要？

```text
NAT 仍然需要多个公网 IP ❌
```

------

### 📌 解决方案

```text
IP + 端口号
```

------

### 📊 转换方式

| 私网             | 公网          |
| ---------------- | ------------- |
| 192.168.1.2:5000 | 1.1.1.1:10000 |
| 192.168.1.3:5001 | 1.1.1.1:10001 |

------

### 📌 优势

```text
一个公网 IP → 支持多个设备
```

------

## 十二、整体通信流程（完整链路）

------

```text
主机A
 ↓
判断是否同网段
 ↓
是 → ARP → 直接通信
否 → 网关（路由器）
 ↓
NAT/NAPT 转换
 ↓
公网通信
 ↓
返回 → 反向转换
 ↓
主机A
```

------

## 🔥 面试高频问题（必会）

------

### Q1：如何判断同一网段？

```text
IP & Mask 比较
```

------

### Q2：为什么需要 NAT？

```text
IPv4 地址不足
```

------

### Q3：NAPT 和 NAT 区别？

| 项目   | NAT  | NAPT      |
| ------ | ---- | --------- |
| 映射   | IP   | IP + 端口 |
| 公网IP | 多个 | 一个      |

------

### Q4：为什么 CIDR 更好？

```text
灵活 + 无分类
```

------

## 🚀 最终理解（核心一句话）

```text
IP = 定位网络
MAC = 定位设备
Mask = 划分边界
Router = 跨网转发
NAT = 地址复用
```
