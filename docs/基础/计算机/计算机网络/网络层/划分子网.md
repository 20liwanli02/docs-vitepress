# 子网划分 & 子网掩码 & CIDR（面试级笔记）

------

## 一、为什么需要子网划分？

### ❗ 原始问题（类ful IP）

以 C 类地址为例：

| 类型 | 网络号   | 主机号  |
| ---- | -------- | ------- |
| C 类 | 前 24 位 | 后 8 位 |

👉 最大主机数：

```text
2^8 - 2 = 254
```

------

### ❗ 问题

- 主机数量不灵活（要么 254，要么更大）
- IP 地址浪费严重

------

## 二、子网划分（核心思想）

### 📌 本质

```text
从主机号中“借位” → 作为子网号
```

------

### 📊 示例：借 1 位

原结构：

```text
网络号（24位） + 主机号（8位）
```

划分后：

```text
网络号（24位） + 子网号（1位） + 主机号（7位）
```

------

### 📊 结果

| 项目           | 数值         |
| -------------- | ------------ |
| 子网数量       | 2¹ = 2       |
| 每个子网主机数 | 2⁷ - 2 = 126 |

------

### 📌 规律总结

| 借位数 | 子网数 | 每网主机数  |
| ------ | ------ | ----------- |
| n      | 2ⁿ     | 2^(8-n) - 2 |

------

## 三、子网掩码（Subnet Mask）

------

### 📌 为什么需要？

👉 子网划分后：

```text
网络号长度不固定 ❌
```

👉 必须引入子网掩码来标识：

```text
哪些是网络位，哪些是主机位
```

------

### 📌 定义

```text
1 → 网络号（含子网号）
0 → 主机号
```

------

### 📊 示例

#### /25 网络

```text
11111111.11111111.11111111.10000000
```

👉 转换为十进制：

```text
255.255.255.128
```

------

## 四、默认子网掩码（重要基础）

| 类别 | 默认掩码      |
| ---- | ------------- |
| A 类 | 255.0.0.0     |
| B 类 | 255.255.0.0   |
| C 类 | 255.255.255.0 |

------

👉 特点：

```text
未划分子网时使用
```

------

## 五、网络地址计算（核心技能）

------

### 📌 方法

```text
IP地址 AND 子网掩码 = 网络地址
```

------

### 📊 示例

IP：

```text
192.168.1.250
```

子网掩码：

```text
255.255.255.128 (/25)
```

------

### 📌 计算

```text
11111010 (250)
AND
10000000 (128)
=
10000000 → 128
```

------

### ✅ 结果

```text
网络地址 = 192.168.1.128
```

------

### 📌 主机号

```text
250 - 128 = 122
```

👉 第 122 台主机

------

## 六、CIDR 表示法（重点）

------

### 📌 定义

```text
IP/网络位长度
```

------

### 📊 示例

```text
192.168.1.250/25
```

👉 表示：

- 前 25 位是网络号
- 后 7 位是主机号

------

### 📌 优点

| 优点 | 说明           |
| ---- | -------------- |
| 简洁 | 不用写子网掩码 |
| 灵活 | 不依赖 ABC 类  |
| 标准 | 现代网络通用   |

------

### 📌 本质

```text
CIDR = IP + 子网掩码的简化表示
```

------

## 七、重要结论（面试必考）

------

### ❗ 结论 1

```text
只知道 IP 地址 ❌
必须知道 子网掩码 或 CIDR
```

------

### ❗ 结论 2

```text
网络划分已不再依赖 ABC 分类
```

------

## 八、是否同一网段判断（超级重点）

------

## 📌 正确判断方法

👉 发送方执行：

```text
1. 自己IP & 自己掩码 → 本地网络
2. 目标IP & 自己掩码 → 目标网络（视角）
3. 对比两者
```

------

### ❗ 关键点（易错）

```text
使用“发送方的子网掩码”！
```

------

## 九、案例分析（经典面试题）

------

### 📊 场景

| 主机 | IP          | CIDR |
| ---- | ----------- | ---- |
| A    | 200.10.10.1 | /24  |
| B    | 200.10.10.2 | /16  |

------

------

## 🧠 情况一：A → B

------

### Step 1：A 计算自己网络

```text
200.10.10.1 & /24 → 200.10.10.0
```

------

### Step 2：A 计算 B 网络（用自己的掩码）

```text
200.10.10.2 & /24 → 200.10.10.0
```

------

### ✅ 结论

👉 A 认为：

```text
同一网络 ✅
```

👉 可以发送

------

------

## 🧠 情况二：B → A

------

### Step 1：B 计算自己网络

```text
200.10.10.2 & /16 → 200.10.0.0
```

------

### Step 2：B 计算 A 网络

```text
200.10.10.1 & /16 → 200.10.0.0
```

------

### ✅ 结论

👉 B 认为：

```text
同一网络 ✅
```

👉 可以发送

------

⚠️ 但现实中你实验的情况不同，是因为：

👉 **没有网关 + ARP失败 + 回包失败**

------

## 🔥 你实验中的核心问题

------

### ❗ 不对称通信原因

| 原因         | 解释           |
| ------------ | -------------- |
| 子网掩码不同 | 判断逻辑不一致 |
| 无默认网关   | 无法跨网段     |
| ARP 失败     | 无法获取 MAC   |
| 回包失败     | 单向通信       |

------

## 十、通信本质逻辑

------

### 📌 判断流程

```text
是否同网段？
    ↓
是 → 直接 ARP → 通信
否 → 发给网关 → 路由器转发
```

------

## 十一、核心总结（面试背诵）

------

### ⭐ 子网划分本质

```text
借主机位 → 划分网络
```

------

### ⭐ 子网掩码本质

```text
标识网络位和主机位
```

------

### ⭐ CIDR 本质

```text
IP + 网络位长度
```

------

### ⭐ 同网段判断

```text
用发送方掩码判断！
```

------

### ⭐ 通信核心

```text
同网段 → 直接通信
不同网段 → 必须网关
```

------

## 🔥 高频面试题

------

### Q1：为什么要子网划分？

👉：

- 提高 IP 利用率
- 灵活划分网络

------

### Q2：CIDR 为什么取代 ABC？

👉：

- 更灵活
- 支持变长子网

------

### Q3：为什么必须有子网掩码？

👉：

```text
IP 本身无法区分网络/主机
```

------

### Q4：如何判断同一网段？

👉：

```text
IP & Mask 比较
```

------

### Q5：为什么会出现“能发不能回”？

👉：

```text
双方网络判断不一致
```

------

## 🚀 最终理解（关键一句话）

```text
网络通信的本质 = MAC（局域网） + IP（跨网络） + 掩码（边界）
```
