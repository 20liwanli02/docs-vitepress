# 一、IP 地址的定义

IP 地址（Internet Protocol Address）是用于在因特网上**唯一标识主机或路由器接口**的逻辑地址。

### 作用

- 标识网络中的设备（主机 / 路由器接口）
- 用于实现跨网络通信与寻址
- 支撑数据从源主机到目标主机的传输路径选择

------

# 二、IP 地址 vs MAC 地址（核心对比）

| 项目     | IP 地址                | MAC 地址                       |
| -------- | ---------------------- | ------------------------------ |
| 层级     | 网络层                 | 数据链路层                     |
| 作用范围 | 跨网络通信（全网寻址） | 局域网内通信                   |
| 是否可变 | 可配置、可变化         | 固定（通常写入网卡）           |
| 功能     | “导航作用”（路由选择） | “本地定位”（同一网络内找设备） |

------

## 通信过程理解（重点逻辑）

跨网络通信基本流程：

1. 源主机 → 通过 **MAC 地址** 找到本网络网关（路由器）
2. 路由器 → 依据 **IP 地址** 进行跨网络转发（路由选择）
3. 到达目标网络后 → 再用 **MAC 地址** 定位目标主机

👉 结论：

- IP：负责“走哪条路”（路径选择）
- MAC：负责“具体找谁”（局域网定位）

------

# 三、IP 地址版本

## 1. IPv4

- 长度：32 位（二进制）
- 表示能力：约 2³² ≈ 42 亿个地址
- 现实问题：地址资源不足（已接近耗尽）

### IPv4 示例

```
11000000 10101000 00000001 00000001
```

------

## IPv4 地址表达方式

### 点分十进制（标准表示法）

将 32 位分成 4 组，每组 8 位（二进制 → 十进制）：

```
192.168.1.1
```

👉 优点：

- 可读性强
- 易于记忆与配置

------

## 2. IPv6

- 长度：128 位
- 地址数量：极其庞大（约 3×10³⁸）
- 目的：彻底解决 IPv4 地址不足问题

👉 现实状态：

- IPv6 已存在，但 IPv4 仍是主流

------

# 四、IP 地址结构（重点考点）

IP 地址 = **网络号 + 主机号**

------

## 1. 网络号（Network ID）

### 作用

- 标识设备所在的网络（或网段）

### 判断同网段规则

👉 如果两个 IP 的网络号相同，则它们属于同一个网络

------

## 2. 主机号（Host ID）

### 作用

- 标识同一网络内的具体设备

### 特点

- 同一网络内主机号必须唯一
- 不同网络之间可以重复

------

## 类比理解

- 网络号 = 小区名称 / 楼栋号
- 主机号 = 门牌号

👉 规则：

- 同一小区不能有重复门牌号
- 不同小区可以重复门牌号（如 101）

------

# 五、网络划分的本质

IP 地址通过“网络号 + 主机号”的划分，实现：

### 1. 局域网（LAN）

- 同一网络号
- 设备直接通过 MAC 通信

### 2. 跨网络通信

- 不同网络号
- 需要路由器转发

### 3. 互联网（Internet）

- 由大量局域网通过路由器互联构成

------

# 六、关键理解总结（非常重要）

### 1. IP 地址本质

👉 是“网络位置 + 设备编号”的组合

------

### 2. 网络层核心职责

- 跨网络寻址（IP）
- 路由选择（Router）

------

### 3. 数据链路层 vs 网络层分工

| 层级              | 作用         |
| ----------------- | ------------ |
| 数据链路层（MAC） | 局域网内传输 |
| 网络层（IP）      | 跨网络传输   |

------

# 七、一句话总结（考试用）

IP 地址是用于在网络层标识主机或路由器接口的逻辑地址，由网络号和主机号组成，用于实现跨网络通信与路由选择。
