# MAC地址与数据链路层工作机制笔记

## 一、MAC 地址的基本概念

### 1️⃣ 什么是 MAC 地址

> **MAC 地址（Media Access Control Address）= 物理地址**

作用：

- **唯一标识一个设备（主机 / 路由器接口 / 网卡）**

------

### 2️⃣ MAC 地址结构

- 长度：**48 位（6 字节）**

```text
XX-XX-XX-XX-XX-XX（每个XX为1字节）
```

说明：

- 每个字节 = 8 位
- 每个十六进制数 = 4 位二进制
- 2 个十六进制 = 1 字节

------

## 二、MAC 地址的存储位置

👉 存在于：

- **网卡（NIC）中**
- 存储在：
  - ROM（只读存储器）

------

### 特点：

- 出厂时写入
- 理论上**全球唯一**
- 用于标识设备

------

## 三、MAC 地址是否可以修改？

👉 结论：

| 类型               | 是否可变 |
| ------------------ | -------- |
| 网卡硬件中的 MAC   | ❌ 不可变 |
| 操作系统使用的 MAC | ✔ 可修改 |

------

### 原理：

- 操作系统默认：
  - 读取网卡中的 MAC
- 也可以：
  - 使用**自定义 MAC 地址**

------

### 修改方式（Windows 示例）：

路径：

```text
网络状态 → 更改适配器 → 属性 → 配置 → 高级 → 本地管理地址
```

说明：

- 不设置 → 使用网卡原始 MAC
- 设置值 → 使用自定义 MAC

------

## 四、广播 MAC 地址（重点）

```text
FF-FF-FF-FF-FF-FF
```

------

### 含义：

> **发送给当前网络内所有主机**

------

### 特点：

- 所有设备都会接收
- 不进行 MAC 比对

------

### 使用场景：

- 局域网广播通信
- ARP 请求（后面会学）

------

## 五、MAC 地址如何工作（核心）

## 1️⃣ 发送端流程

```text
数据 → 网络层 → 数据链路层（加 MAC 头）→ 物理层 → 发送
```

------

### 数据链路层添加内容：

- 源 MAC 地址
- 目标 MAC 地址
- 帧尾（校验）

------

## 2️⃣ 接收端流程

```text
接收 → 物理层 → 数据链路层 → 判断 MAC → 决定是否接收
```

------

## 六、MAC 地址的判断机制（重点）

### 接收端网卡行为：

```text
目标MAC == 自己的MAC ?
```

------

### 情况 1：相等

✔ 接收数据
✔ 上交上层处理

------

### 情况 2：不相等

❌ 丢弃数据

------

### 情况 3：广播 MAC

```text
FF-FF-FF-FF-FF-FF
```

✔ 所有主机都接收

------

## 七、网卡的作用（重要）

👉 网卡工作在：

```text
数据链路层 + 物理层
```

------

### 网卡负责：

1. 发送数据（信号转换）
2. 接收数据（信号还原）
3. 解析链路层：
   - MAC 地址
   - 校验信息
4. 决定是否接收数据

------

## 八、完整数据流（结合 MAC）

### 发送端：

```text
应用层 → 运输层 → 网络层 → 数据链路层（加MAC）→ 物理层 → 发送
```

------

### 接收端：

```text
接收 → 物理层 → 数据链路层（判断MAC）→ 网络层 → 运输层 → 应用层
```

------

## 九、核心理解总结

### 1️⃣ MAC 地址的本质

> **局域网内唯一标识设备**

------

### 2️⃣ 数据链路层的核心作用

> **通过 MAC 地址实现“同一链路内通信”**

------

### 3️⃣ 接收原则

- 发给我 → 收
- 不是我 → 丢
- 广播 → 全收

------

### 4️⃣ 关键一句话

> **MAC 地址决定“这一段链路中谁该接收数据”**

------

## 十、补充理解（非常重要）

👉 你可以这样理解：

- **MAC 地址 → 解决“这一跳给谁”**
- **IP 地址 → 解决“最终给谁”**
