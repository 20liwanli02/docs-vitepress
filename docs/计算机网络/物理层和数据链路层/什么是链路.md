# 什么是链路

## 一、数据链路层的位置

- 位于五层模型中的**第二层（自下而上）**
- 上一层：网络层
- 下一层：物理层

分层结构：

```text
应用层 → 运输层 → 网络层 → 数据链路层 → 物理层
```

------

## 二、物理层 vs 数据链路层（回顾对比）

### 物理层解决：

> **比特（0/1） ↔ 信号（波形）转换**

------

### 数据链路层解决：

> **数据如何在“一个网络 / 一段链路”内传输**

------

## 三、什么是“链路”和“网络”（重点）

### 1️⃣ 链路（Link）

👉 定义：

> 从一个节点到**相邻节点**的一段物理线路

------

### 节点包括：

- 主机（电脑、服务器）
- 路由器
- 交换机
- 集线器

------

### 关键特征：

- 中间**没有路由器**
- 可以是：
  - 有线（网线）
  - 无线（WiFi）

------

## 四、什么是“同一链路 / 同一网络”

👉 判断标准：

> **中间是否经过路由器**

------

### ✔ 同一链路（同一网络）

- 节点之间**没有路由器**
- 属于同一个局域网（LAN）

例如：

- 主机 ↔ 主机
- 主机 ↔ 路由器接口

------

### ❌ 不同链路（不同网络）

- 中间**经过路由器**
- 属于不同网络

------

## 五、案例理解（核心）

假设网络结构：

```text
主机A ─┬─ 主机B
        │
        └─ 主机C ─ 路由器1 ─ 路由器2 ─ 主机D
```

------

### 分析：

#### 同一链路（同一网络）：

- 主机A、B、C之间 ✔
- 主机C 与 路由器1接口 ✔

------

#### 不同链路（不同网络）：

- 主机A → 主机D ❌（跨多个路由器）

------

## 六、数据链路层的职责（核心）

👉 只负责：

```text
一个网络（链路）内的数据传输
```

------

### 举例：

#### 情况1：同一网络通信

```text
主机A → 主机B
```

✔ 数据链路层可以**直接完成**

------

#### 情况2：跨网络通信

```text
主机A → 主机D
```

流程：

1. 数据链路层：
   - A → 路由器1（只负责这一段）
2. 网络层：
   - 决定路径（路由）
3. 数据链路层：
   - 路由器1 → 路由器2
   - 路由器2 → 主机D

------

## 七、链路层 vs 网络层（关键区别）

| 层         | 作用                          |
| ---------- | ----------------------------- |
| 数据链路层 | 负责**一段链路内传输**        |
| 网络层     | 负责**跨网络传输 + 路由选择** |

------

## 八、可以这样理解（非常重要）

👉 类比“快递运输”

------

### 数据链路层：

> **负责“这一段路怎么走”**

类似：

- 一个配送站 → 下一个配送站

------

### 网络层：

> **负责“整体路线规划”**

类似：

- 从北京 → 上海 的整体路径

------

## 九、总结

### 核心一句话

> **数据链路层只管局域网（单个链路），网络层才负责跨网络通信**

------

### 关键记忆点

1. 链路 = 相邻节点之间的物理线路
2. 是否同一网络 → 看是否经过路由器
3. 数据链路层：
   - 只管“一段路”
4. 网络层：
   - 负责“全局路径”

------
