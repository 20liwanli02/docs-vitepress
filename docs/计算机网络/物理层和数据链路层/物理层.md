# 物理层笔记

## 一、物理层的位置

- 物理层位于计算机网络分层体系的**最底层**
- 上一层是：**数据链路层**

分层结构（自上而下）：

```text
应用层 → 运输层 → 网络层 → 数据链路层 → 物理层
```

------

## 二、物理层的核心作用

👉 一句话总结：

> **解决“用什么信号表示 0 和 1”的问题**

------

## 三、如何理解“信号表示比特”

### 1️⃣ 数据本质

- 所有数据本质都是：

```text
0101（二进制）
```

------

### 2️⃣ 为什么需要信号？

👉 因为：

- 网络中不能直接传输“0 和 1”
- 必须通过物理介质传播

------

### 3️⃣ 信号形式

物理层将比特转换为：

- 电信号（网线）
- 光信号（光纤）
- 电磁波（无线）

------

## 四、物理层的核心工作流程

### 发送端

```text
比特流（0101） → 转换为信号（波形） → 在介质中传播
```

步骤：

1. 接收来自数据链路层的数据
2. 添加物理层信息（协议相关）
3. 将比特流转换为信号（编码/调制）
4. 发送到传输介质

------

### 接收端

```text
信号（波形） → 还原为比特流（0101）
```

步骤：

1. 接收信号
2. 按协议解析信号
3. 还原出比特流
4. 去掉物理层信息
5. 交给数据链路层

------

## 五、发送与接收的整体流程

### 发送端（自上而下）

```text
上层数据 → 数据链路层封装 → 物理层转换为信号 → 发送
```

------

### 接收端（自下而上）

```text
接收信号 → 物理层还原比特 → 数据链路层处理 → 上层处理
```

------

## 六、物理层与数据链路层关系

- 数据链路层：
  - 负责**帧的封装（头 + 尾）**
  - 负责**差错检测**
- 物理层：
  - 不关心数据内容
  - 只负责：

```text
比特流 ↔ 信号
```

------

## 七、协议的重要性

👉 物理层通信必须满足：

> **发送端和接收端使用相同协议**

否则：

- 无法正确解析信号
- 无法还原比特

------

## 八、关键理解点

### 1️⃣ 物理层处理的仍然是二进制

- 从应用层开始就是 0101
- 物理层只是“换表现形式”

------

### 2️⃣ 本质是“映射关系”

```text
0 / 1 ↔ 某种信号形式
```

------

### 3️⃣ 物理层不关心内容

- 不关心：
  - HTTP
  - TCP
  - IP
- 只关心：
  - 如何传输比特

------

## 九、物理层提供的服务

👉 提供：

> **比特流传输服务**

通俗理解：

- 把 0101 变成可以在介质上传播的信号
- 再把信号还原成 0101

------

## 十、总结

### 核心流程

```text
发送端：比特 → 信号  
接收端：信号 → 比特
```

------

### 核心一句话

> **物理层的本质就是实现“比特流在物理介质上的传输”**
