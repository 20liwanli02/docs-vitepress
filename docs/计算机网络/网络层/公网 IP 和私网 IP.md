# 🌍 公网 IP、私网 IP 与 NAT / NAPT（面试重点）

## 一、为什么需要公网 IP 和私网 IP？

### 背景问题

- IPv4 地址总数有限（约 43 亿）
- 互联网设备数量远超这个数量

👉 需要**缓解 IP 地址不足问题**

------

## 二、公网 IP vs 私网 IP

### 1️⃣ 公网 IP（Public IP）

- 用于：**互联网（广域网）通信**
- 特点：
  - 🌐 全球唯一
  - 🚫 不可重复

📌 示例：

```text
8.8.8.8
114.114.114.114
```

------

### 2️⃣ 私网 IP（Private IP）

- 用于：**局域网内部（LAN）**
- 特点：
  - ✅ 可以重复
  - ❌ 不能直接访问公网

------

### 📊 私网 IP 范围（必考）

| 类别 | 范围                          |
| ---- | ----------------------------- |
| A类  | 10.0.0.0 ~ 10.255.255.255     |
| B类  | 172.16.0.0 ~ 172.31.255.255   |
| C类  | 192.168.0.0 ~ 192.168.255.255 |

📌 常见：

```text
192.168.1.1（家庭路由器）
```

------

## 三、核心思想（⭐⭐）

👉 **公网 IP + 私网 IP 的组合：**

```text
局域网内部 → 使用私网 IP（可重复）
访问互联网 → 使用公网 IP（唯一）
```

------

## 四、关键问题

❓ 不同局域网中：

```text
192.168.1.1（你家）
192.168.1.1（别人家）
```

👉 如何通信？

------

## 五、NAT（网络地址转换）

### 1️⃣ 定义

👉 **NAT = Network Address Translation**

```text
私网 IP ⇄ 公网 IP
```

------

### 2️⃣ 工作原理

#### 核心设备：

👉 **NAT 路由器（网关）**

------

### 3️⃣ 转换流程（面试重点）

#### 📤 发送过程：

```text
私网主机 → 网关（NAT路由器）
→ 替换源IP（私网 → 公网）
→ 发往互联网
```

------

#### 📥 接收过程：

```text
公网返回数据 → NAT路由器
→ 查 NAT 表
→ 转换为私网 IP
→ 转发给内网主机
```

------

### 4️⃣ NAT 映射表（关键）

| 私网 IP      | 公网 IP     |
| ------------ | ----------- |
| 192.168.1.10 | 202.100.1.1 |

------

### 5️⃣ NAT 优缺点

#### ✅ 优点：

- 节省公网 IP

#### ❌ 缺点：

- 仍需多个公网 IP（IP 池）
- 扩展性有限

------

## 六、NAPT（重点 ⭐⭐⭐）

### 1️⃣ 定义

👉 **NAPT = Network Address Port Translation**

```text
IP + 端口号 的转换
```

------

### 2️⃣ 核心思想

👉 **多个私网主机共享一个公网 IP**

通过：

```text
端口号区分不同主机/进程
```

------

### 3️⃣ 映射表（关键）

| 私网地址          | 公网地址         |
| ----------------- | ---------------- |
| 192.168.1.10:5000 | 202.100.1.1:3000 |
| 192.168.1.11:5001 | 202.100.1.1:3001 |

------

### 4️⃣ 工作流程

#### 📤 发送：

```text
私网IP:端口 → NAT路由器
→ 转换为 公网IP:新端口
→ 发往互联网
```

------

#### 📥 接收：

```text
公网IP:端口 → NAT路由器
→ 查 NAPT 表
→ 转换为 私网IP:端口
→ 转发
```

------

### 5️⃣ 为什么只需要一个公网 IP？

👉 因为：

```text
IP 相同，但端口不同 → 可唯一标识通信
```

------

## 七、NAT vs NAPT（面试对比）

| 对比项       | NAT    | NAPT      |
| ------------ | ------ | --------- |
| 转换内容     | IP     | IP + 端口 |
| 公网 IP 数量 | 多个   | 一个即可  |
| 使用场景     | 企业级 | 家庭/主流 |
| 节省 IP      | 一般   | 极强      |

------

## 八、现实应用（面试加分）

### 🏠 家庭网络

- 路由器类型：
  👉 **NAPT 路由器**
- 一个公网 IP 支持：
  - 手机
  - 电脑
  - 平板
  - IoT设备

------

## 九、核心流程总结（必背）

### 🌐 内网访问公网

```text
主机 → 网关
→ NAPT转换（IP+端口）
→ 公网通信
```

------

### 🌐 公网返回数据

```text
服务器 → 公网IP
→ NAT路由器
→ 查表
→ 转发到内网主机
```

## 十一、一句话总结（面试杀招）

👉 **NAT / NAPT 本质：**

> 通过“地址转换（IP）”和“端口复用（Port）”，让大量私网设备共享有限的公网 IP，实现互联网通信。

## 十二、补充（面试加分）

### ⚠️ NAT 的问题

- 破坏端到端通信（P2P困难）
- 需要端口映射（如内网穿透）
