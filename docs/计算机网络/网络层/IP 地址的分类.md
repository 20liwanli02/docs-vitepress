# 一、IP 地址分类（Classful Addressing）

## 1. 分类概览

IP 地址（IPv4）共分为 5 类：

| 类别 | 用途              | 是否可分配 |
| ---- | ----------------- | ---------- |
| A 类 | 大型网络          | ✅          |
| B 类 | 中型网络          | ✅          |
| C 类 | 小型网络          | ✅          |
| D 类 | 多播（Multicast） | ❌          |
| E 类 | 保留地址          | ❌          |

👉 实际需要重点掌握：**A / B / C 类**

------

## 2. 分类依据（看前几位）

| 类别 | 二进制前缀 | 十进制范围 |
| ---- | ---------- | ---------- |
| A 类 | 0          | 1 ~ 126    |
| B 类 | 10         | 128 ~ 191  |
| C 类 | 110        | 192 ~ 223  |

------

# 二、网络号 & 主机号

| 类别 | 网络号 | 主机号 |
| ---- | ------ | ------ |
| A 类 | 8 位   | 24 位  |
| B 类 | 16 位  | 16 位  |
| C 类 | 24 位  | 8 位   |

------

# 三、特殊地址（必须掌握）

## 1. 网络地址（Network Address）

- 特点：**主机号全 0**
- 作用：表示一个网络
- ❌ 不能分配给主机

例：

```
192.168.1.0
```

------

## 2. 广播地址（Broadcast Address）

- 特点：**主机号全 1**
- 作用：向该网络所有主机发送数据
- ❌ 不能分配

例：

```
192.168.1.255
```

------

## 3. 可用主机范围

```
192.168.1.1 ~ 192.168.1.254
```

👉 公式：

```
可用主机数 = 2^n - 2
```

（减去 网络地址 + 广播地址）

------

# 四、示例分析

## IP：192.168.1.1

### 1. 判断类别

```
192 → 二进制 110xxxxx → C 类
```

### 2. 结构

```
网络号：192.168.1
主机号：1
```

### 3. 推导

| 项目     | 地址                        |
| -------- | --------------------------- |
| 网络地址 | 192.168.1.0                 |
| 广播地址 | 192.168.1.255               |
| 可用范围 | 192.168.1.1 ~ 192.168.1.254 |

------

# 五、多播 & 广播

## 1. 单播（Unicast）

- 一对一通信

## 2. 多播（Multicast）

- 一对多（组播）
- 使用 **D 类地址**
- 一个地址 → 一组主机

## 3. 广播（Broadcast）

- 一对所有
- 是多播的特殊情况

------

# 六、各类地址规模

## 1. A 类

- 网络号：7 位（首位固定为 0）

👉 网络数量：

```
2^7 - 2 = 126
```

👉 主机数量：

```
2^24 - 2 ≈ 1677 万
```

👉 范围：

```
1.0.0.0 ~ 126.255.255.255
```

------

## 2. B 类

- 网络号：14 位（前两位固定 10）

👉 网络数量：

```
2^14 = 16384
```

👉 主机数量：

```
2^16 - 2 = 65534
```

👉 范围：

```
128.0.0.0 ~ 191.255.255.255
```

------

## 3. C 类

- 网络号：21 位（前三位固定 110）

👉 网络数量：

```
2^21 ≈ 200 万
```

👉 主机数量：

```
2^8 - 2 = 254
```

👉 范围：

```
192.0.0.0 ~ 223.255.255.255
```

------

# 七、特殊说明

## 1. 为什么 A 类是 1~126？

- `0.x.x.x` → 表示当前网络
- `127.x.x.x` → 回环地址（Loopback）

典型：

```
127.0.0.1 → 本机测试地址
```

------

# 八、分类方式的缺陷（重点）

## 问题：**IP 地址浪费严重**

### 示例：

- 公司只有 20 台电脑
- 却申请了 C 类（254 个地址）

👉 浪费 200+ IP

------

## 根本原因：

👉 **网络号 & 主机号位数是固定的**

| 类别 | 主机容量 |
| ---- | -------- |
| A 类 | 超大     |
| B 类 | 很大     |
| C 类 | 较小     |

👉 不能灵活分配

------

# 九、解决方案（引出后续知识）

👉 为了解决浪费问题，引入：

### **CIDR（无分类编址）**

核心思想：

```
网络号长度可变（不再固定 A/B/C）
```

例如：

```
192.168.1.0/26
```

👉 可按需分配 IP

------
