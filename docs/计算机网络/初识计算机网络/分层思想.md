# 分层思想

## 一、仅有 HTTP 协议头还不够

在前面的分析中，我们已经知道：
- 数据在发送前会被封装
- 在原始数据（Body）前添加 **协议头（Header）**
- 形成一个完整的 **协议数据单元（PDU）**

例如：

[ HTTP Header | Body ]

但问题来了：

👉 **这样就可以直接发送了吗？答案：不行**

## 二、通信的本质：进程与进程之间通信

虽然我们常说：

> 主机 A 和 主机 B 通信

但实际上：

> ❗真正通信的是 **进程（Process）**

### 什么是进程？

- 正在运行的程序（在内存中，被 CPU 执行）
- 例如：
  - 浏览器（发送请求）
  - Web 服务器（接收请求）

📌 通信模型：

浏览器进程（A主机） ←→ Web服务器进程（B主机）

## 三、新问题：如何标识“哪个进程”？

当前问题：

👉 数据发过去了，但 **不知道交给哪个进程处理**

### 解决方案：端口号（Port）

- 端口号 = 进程的标识符（逻辑）
- 操作系统维护：

进程 ←→ 端口号（唯一对应）

### 示例：

| 进程      | 端口号 |
| --------- | ------ |
| 浏览器    | 52345  |
| Web服务器 | 80     |


## 四、错误设计：把端口号写进 HTTP 头？

直觉做法：

HTTP Header:

- srcPort: xxx
- dstPort: 80

❌ 问题：

- 不止 HTTP 需要端口号
- FTP / DNS / SMTP 都需要
- 会导致：
  - 重复代码
  - 冗余设计
  - 协议耦合严重

## 五、正确设计：抽象出“传输层协议”

![alt text](/images/computer_network/layered_thinking/01.webp)

类似软件工程中的：

> ❗“提取公共逻辑”

👉 网络中做法：

- 把“端口功能”抽出来
- 形成新的协议层

### 这个协议就是：

- **TCP**
- **UDP**

## 六、封装过程（分层体现）

### 1️⃣ 应用层（HTTP）

[ HTTP Header | Body ]

↓

### 2️⃣ 传输层（TCP）

- 把 HTTP 整体当作数据（Body）
- 再加一层头

[ TCP Header (端口号) | HTTP Header | Body ]

📌 TCP Header 包含：
- 源端口（src port）
- 目的端口（dst port）



## 七、接收端的解封装过程

### 步骤：

1. 接收 TCP 数据
2. 解析 TCP Header，找到目标端口（如 80 的进程）
3. 定位进程（Web服务器，必须有 HTTP 协议能力）
4. 去掉 TCP Header
5. 把数据交给 HTTP 协议处理
6. HTTP 再解析 Header
7. 得到最终原始数据



## 八、为什么需要源端口？

问题：

> 只传目标端口不行吗？

答案：❌ 不行

### 原因：

👉 用于“回信”

客户端 → 服务器：
src: 52345
dst: 80

服务器 → 客户端：
src: 80
dst: 52345

📌 源端口 = “回信地址”



## 九、分层思想（核心）

### 典型分层结构：

应用层：HTTP / FTP / DNS
↓
传输层：TCP / UDP
↓
网络层：IP



## 十、各层作用总结

### 1️⃣ 应用层

- 面向用户
- 定义数据格式（HTTP / JSON 等）



### 2️⃣ 传输层（TCP / UDP）

👉 提供：

- **进程 → 进程通信（端到端）**

#### TCP：

- 可靠传输（确认、重传、顺序）

#### UDP：

- 不可靠（快、无连接）



### 3️⃣ 网络层（IP）

👉 提供：

- **主机 → 主机通信**

解决问题：

- 数据从哪台机器 → 到哪台机器



## 十一、为什么要分层？

### 优点一：降低复杂度

❌ 不分层：

- 所有逻辑混在一起
- 极其复杂

✅ 分层：

- 各层职责清晰
- 易维护



### 优点二：问题隔离

- 哪一层出问题 → 修哪一层
- 不影响其他层



### 优点三：可独立演化

#### 示例：

- IPv4 → IPv6

👉 不影响：
- TCP
- HTTP

📌 原因：

> 上层只关心“服务”，不关心实现


## 十二、核心总结

### 数据传输完整流程：

应用层（HTTP）
↓
传输层（TCP：加端口）
↓
网络层（IP：找主机）
↓
发送
↓
接收端逐层解封装



### 核心概念记忆：

- 协议 = 约定
- 端口 = 进程标识
- TCP/UDP = 解决进程通信
- IP = 解决主机通信
- 分层 = 降低复杂度 + 解耦



## ✅ 一句话总结

> 计算机网络的本质：**通过分层协议，实现从一个进程到另一个进程的可靠数据传输**
