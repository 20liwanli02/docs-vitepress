# 分层体系结构

## 一、网络分层模型

![alt text](/images/computer_network/layered_thinking/02.png)

### 1. OSI 参考模型（7层）

> **理论标准（纸面标准）**

从下到上共 7 层：

1. 物理层
2. 数据链路层
3. 网络层
4. 运输层
5. 会话层
6. 表示层
7. 应用层

👉 特点：

* 结构完整、定义清晰
* **实际应用中使用较少**

### 2. TCP/IP 参考模型（4层）

> **事实标准（实际使用）**

| TCP/IP层 | 对应OSI层          |
| ------- | --------------- |
| 应用层     | 应用层 + 表示层 + 会话层 |
| 运输层     | 运输层             |
| 网际层     | 网络层             |
| 网络接口层   | 数据链路层 + 物理层     |

👉 命名原因：

* **TCP**：运输层核心协议
* **IP**：网际层核心协议（Internet Protocol）

👉 特点：

* 实际互联网使用标准
* 更加简化

### 3. 教学五层模型（重点）

> **学习使用**

1. 应用层
2. 运输层
3. 网络层
4. 数据链路层
5. 物理层

👉 本课程基于此模型


## 二、各层作用（核心理解）

### 1. 应用层

👉 **解决：应用程序通信问题**

常见协议：

* HTTP（网页）
* FTP（文件传输）
* SMTP（邮件）

👉 本质：

> 面向用户应用


### 2. 运输层

👉 **解决：进程 ↔ 进程 通信（端到端）**

常见协议：

* TCP
* UDP

### 3. 网络层

👉 **解决：主机 ↔ 主机通信 + 路由**

功能：

* IP 寻址
* 路由选择

👉 通俗理解：

> 数据如何跨网络，从一个主机到另一个主机

### 4. 数据链路层

👉 **解决：局域网内传输问题**

功能：

* 帧封装
* MAC 地址识别
* 差错检测

👉 范围：

> 一段链路 / 一个网络（局域网）

### 5. 物理层

👉 **解决：比特如何表示（0/1）**

例如：

* 电信号
* 光信号
* 无线电波

## 三、数据封装过程（发送端）

![alt text](/images/computer_network/layered_thinking/QQ_1775042934898.png)

### 总体流程

👉 **从上到下逐层封装**

```
应用层 → 运输层 → 网络层 → 数据链路层 → 物理层
```

### 数据单位变化

| 层     | 名称          |
| ----- | ----------- |
| 应用层   | 报文（Message） |
| 运输层   | 段（Segment）  |
| 网络层   | 包（Packet）   |
| 数据链路层 | 帧（Frame）    |
| 物理层   | 比特流（Bit）    |


### 封装过程

1. 应用层：添加应用层协议头
2. 运输层：添加 TCP/UDP 头
3. 网络层：添加 IP 头
4. 数据链路层：

   * 添加头 + 尾（校验）
5. 物理层：

   * 转换为信号（电/光/无线）

👉 本质：

> 一层一层“加壳”

## 四、数据解封过程（接收端）

![alt text](/images/computer_network/layered_thinking/QQ_1775043064392.png)

### 总体流程

👉 **从下到上逐层解封**

```
物理层 → 数据链路层 → 网络层 → 运输层 → 应用层
```

### 解封过程

1. **物理层**

   * 信号 → 比特流
   * 去掉物理层信息

2. **数据链路层**

   * 校验数据（CRC）
   * 判断 MAC 地址
   * 去头去尾

3. **网络层**

   * 判断 IP 地址
   * 去掉 IP 头

4. **运输层**

   * 解析端口号
   * 分发给对应进程

5. **应用层**

   * 按协议解析数据（如 HTTP）

## 五、关键概念总结

### 1. 协议（Protocol）

👉 本质：

> 通信双方的“约定规则”

作用：

* 定义数据格式
* 定义通信流程

### 2. 分层思想

优点：

* 降低复杂度
* 各层独立发展
* 易于维护与排错

👉 类比：

> 各司其职，互不干扰

### 3. 通信必须满足

👉 **对等层协议必须一致**

否则：

* 无法解析数据
* 通信失败

### 4. 数据包完整性原则（重点）

👉 网络上传输的数据包：

✔ 可以没有上层
❌ 不能没有下层

例如：

* 可以没有应用层（如 ICMP）
* **必须有链路层 + 物理层**

👉 原因：

> 没有下层就无法在网络上传输

### 5. 两个核心顺序（必记）

#### 发送端：

> **自上而下（封装）**

#### 接收端：

> **自下而上（解封）**

## 六、核心一句话总结

👉 **计算机网络本质：**

> 数据在分层协议体系中
> **逐层封装 → 网络传输 → 逐层解封**
