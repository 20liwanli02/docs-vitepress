# 📡 计算机网络入门笔记

## 🎯 学习目标
本节课的目标是：
- 对**计算机网络**形成整体认知
- 理解“**计算机之间如何通信**”的基本原理

---

## 🌐 一、什么是计算机网络？

计算机网络可以将**原本孤立的计算机连接起来**，使它们能够：

- 互相发送数据
- 进行信息交换

### 基本概念：
- **主机（Host）**：接入网络的计算机
- **发送方（Sender）**：发送数据的一方
- **接收方（Receiver）**：接收数据的一方

📌 同一台主机：
- 发送时 → 发送方
- 接收时 → 接收方

---

## 📶 二、数据是如何传输的？

### 1️⃣ 数据的本质形式

计算机网络中，数据不是直接以字符传输，而是：

> **以“信号波形”的形式在线路中传播**

---

### 2️⃣ 数据传输流程

原始数据 → 编码 → 二进制（0/1） → 信号波 → 传输 → 接收 → 解码

---

### 3️⃣ 为什么要编码？

计算机只能识别：

> **二进制（0 和 1）**

因此需要通过**编码方式**将数据转换：

| 数据类型 | 编码方式            |
| -------- | ------------------- |
| 文本     | ASCII / GBK / UTF-8 |
| 图片     | 图像编码            |
| 音视频   | 音视频编码          |

---

## ⚠️ 三、仅传输数据会出现的问题

### ❌ 问题1：无法确定数据边界

例如发送 `"Hi"`：

Hi → 编码 → 01001000 01101001

接收方问题：
- 不知道数据长度
- 不知道从哪里开始、哪里结束

👉 如果多个数据混在一起：

0101 0101 1111 0000 ...

无法区分不同数据！

---

### ❌ 问题2：无法识别数据格式

例如：
- 普通文本 `"Hi"`
- JSON 数据 `{ "name": "Tom" }`

接收方：
- 不知道数据类型
- 不知道该用什么方式解析

---

## 🧠 四、解决方案：添加“额外信息”

### ✔️ 思路

除了发送数据本身，还需要发送：

> **描述数据的信息（元数据）**

---

### 📦 数据结构

+------------+----------------+
| Header | Body |
+------------+----------------+

#### 🔹 Header（首部）

包含：
- 数据长度（Content-Length）
- 数据类型（Content-Type）
- 其他控制信息

#### 🔹 Body（主体 / Payload）
- 实际数据内容

---

## 🤝 五、核心：协议（Protocol）

### 📌 定义

> **协议 = 通信双方共同遵守的规则（约定）**

---

### 🧩 协议解决的问题

协议规定：

| 内容     | 说明               |
| -------- | ------------------ |
| 数据格式 | Header 怎么写      |
| 字段含义 | length 表示什么    |
| 编码方式 | 如何解析数据       |
| 处理方式 | 收到数据后如何处理 |

---

### ⚠️ 关键点

✔ 协议的本质不是格式，而是：

> **双方必须遵守同一套规则**

✔ 必须满足：
- 有协议 ✅
- 双方使用**同一个协议** ✅

否则：
> ❌ 无法通信

---

## 💻 六、协议的实现

协议不是“口头约定”，而是：

> **通过代码实现的规则**

例如：

### 发送端：
- 按协议构造 Header
- 拼接数据
- 发送

### 接收端：
- 解析 Header
- 读取数据长度
- 按类型处理数据

---

## 🌍 七、常见网络协议

| 协议        | 用途               |
| ----------- | ------------------ |
| HTTP        | 超文本传输（网页） |
| FTP         | 文件传输           |
| SMTP        | 邮件发送           |
| POP3 / IMAP | 邮件接收           |

📌 特点：
- 协议很多
- 各自有不同规则
- 针对不同场景优化

---

## 📦 八、协议数据单元（PDU）

### 定义：

> **协议数据单元（Protocol Data Unit） = Header + Body**

PDU = Header + Payload

---

## 🧾 九、总结

### 🔑 核心要点

1. **网络的本质**
    - 让计算机之间可以通信

2. **数据传输过程**
    - 数据 → 二进制 → 信号 → 传输 → 还原

3. **关键问题**
    - 数据边界不清
    - 数据格式不明

4. **解决方案**
    - 添加 Header（额外信息）

5. **最核心概念**
   > ⭐ 协议 = 通信的基础

6. **通信成立条件**
    - 双方必须使用**相同协议**

---

## 🚀 一句话理解

> **计算机网络通信 = 数据 + 额外信息（Header） + 双方遵守的协议**
