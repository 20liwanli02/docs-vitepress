# Redis List 数据结构

## 一、源代码

```java
// 分页查询笔记评论
@Override
public PageResponseDTO<CommentRespDTO> queryInPage(CommentQueryReqDTO commentQueryReqDTO) {
    // 尝试从redis缓存中获取评论数据
    PageResponseDTO<CommentRespDTO> cachePageResponseDTO = this.tryGetFromRedis(commentQueryReqDTO);
    if (cachePageResponseDTO != null) {
        return cachePageResponseDTO;
    }
	// 根据笔记 id 分页查询评论 
    LambdaQueryWrapper<CommentPO> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(CommentPO::getNoteId, commentQueryReqDTO.getNoteId());
    queryWrapper.eq(CommentPO::getStatus, CommentStatusEnum.PUBLISHED.getCode());
    queryWrapper.orderByDesc(CommentPO::getId);
    IPage<CommentPO> commentPOIPage = this.getBaseMapper().selectPage(new Page<>(commentQueryReqDTO.getPage(), commentQueryReqDTO.getPageSize()), queryWrapper);

    if (commentPOIPage.getRecords().isEmpty()) {
        //设置空值缓存
        saveCommentPageInCache(commentQueryReqDTO, null);
        return PageResponseDTO.emptyPage();
    }

    //List结构存储
    List<CommentRespDTO> commentRespDTOS = new ArrayList<>();
    // 收集评论 ids
    List<Long> commentIds = commentPOIPage.getRecords().stream().map(CommentPO::getCommentId).collect(Collectors.toList());
    // 统计每个评论的回复总数
    List<CommentRepliesCountRespDTO> commentRepliesCountList = commentRepliesService.countRepliesCountByCommentId(commentIds);
    // { commentId -> count }
    Map<Long, Integer> repliesCountMap = commentRepliesCountList.stream().collect(Collectors.toMap(CommentRepliesCountRespDTO::getCommentId, CommentRepliesCountRespDTO::getCount));
    // 组装返回数据
    for (CommentPO commentPO : commentPOIPage.getRecords()) {
        CommentRespDTO commentRespDTO = BeanCopyUtils.convert(commentPO, CommentRespDTO.class);
        Integer count = repliesCountMap.get(commentRespDTO.getCommentId());
        commentRespDTO.setRepliesCount(count);
        commentRespDTO.setHasMoreReplies(count != null && count > 0);
        commentRespDTOS.add(commentRespDTO);
    }
    // 写入缓存并返回
    this.saveCommentPageInCache(commentQueryReqDTO, commentRespDTOS);

    PageResponseDTO<CommentRespDTO> commentPageResponse = new PageResponseDTO<>();
    commentPageResponse.setPage(commentQueryReqDTO.getPage());
    commentPageResponse.setSize(commentQueryReqDTO.getPageSize());
    commentPageResponse.setHasNext(commentPOIPage.getTotal() >= (commentQueryReqDTO.getPageSize() * commentQueryReqDTO.getPage()));
    commentPageResponse.setDataList(commentRespDTOS);
    return commentPageResponse;
}

/**
 * 将评论数据保存至redis的list集合中
 *
 * @param commentQueryReqDTO
 * @param pageRespDTOList
 */
private void saveCommentPageInCache(CommentQueryReqDTO commentQueryReqDTO, List<CommentRespDTO> pageRespDTOList) {
    //前置判断，如果目前的分页数过深，则不再做存放
    if (commentQueryReqDTO.getPage() > LIMIT_PAGE_NO) {
        log.info("current visit page is reach cache limit:{}", commentQueryReqDTO.getPage());
        return;
    }
    String noteCommentCacheKey = redisKeyBuilder.getCommentPageCacheKey(commentQueryReqDTO.getNoteId());
    if (pageRespDTOList != null) {
        List<String> jsonCacheList = pageRespDTOList.stream().map(item -> JSON.toJSONString(item)).toList();
        redisTemplate.opsForList().rightPushAll(noteCommentCacheKey, jsonCacheList);
    } else {
        //设置空值缓存
        redisTemplate.opsForList().rightPush(noteCommentCacheKey, NULL_STR);
    }
    redisTemplate.expire(noteCommentCacheKey, 1, TimeUnit.DAYS);
}

/**
 * 尝试从redis缓存中获取评论数据
 *
 * @param commentQueryReqDTO
 * @return
 */
private PageResponseDTO<CommentRespDTO> tryGetFromRedis(CommentQueryReqDTO commentQueryReqDTO) {
    int page = commentQueryReqDTO.getPage();
    int pageSize = commentQueryReqDTO.getPageSize();
    if (page <= 0 || pageSize <= 0) {
        return PageResponseDTO.emptyPage();
    }
    int start = (page - 1) * pageSize;
    //边界值空置
    int end = start + pageSize - 1; // 1 ~ 5
    String noteCommentCacheKey = redisKeyBuilder.getCommentPageCacheKey(commentQueryReqDTO.getNoteId());
    Long len = redisTemplate.opsForList().size(noteCommentCacheKey);
    //命中缓存
    // 先判空
    if (len != null && len > 0) {
        List<String> pageRecords = redisTemplate.opsForList().range(noteCommentCacheKey, start, end);
        //空值缓存
        if (CollectionUtils.isNotEmpty(pageRecords) && NULL_STR.equals(pageRecords.get(0))) {
            return PageResponseDTO.emptyPage();
        }
        if (CollectionUtils.isEmpty(pageRecords)) {
            //可能是后边页的数据暂未有存入缓存
            return null;
        }
        List<CommentRespDTO> dataList = pageRecords.stream().map(item -> JSON.parseObject(item, CommentRespDTO.class)).collect(Collectors.toList());
        PageResponseDTO<CommentRespDTO> commentPageResponse = new PageResponseDTO<>();
        commentPageResponse.setPage(commentQueryReqDTO.getPage());
        commentPageResponse.setSize(commentQueryReqDTO.getPageSize());
        commentPageResponse.setHasNext(len > (commentQueryReqDTO.getPage() * commentQueryReqDTO.getPageSize()));
        commentPageResponse.setDataList(dataList);
        return commentPageResponse;
    }
    return null;
}
```

## 二、代码分析

根据笔记 🆔 分页查询一级评论并且连查评论对应的回复数 countReplies，根据 noteId 设置缓存的 key，使用 list 结构缓存数据，加快
响应速度

设置 Redis 中的分页临界值，(page - 1) * pageSize，索引是从 0 开始的

使用 redis 归根结底就是能更快更稳的获取和从数据库获取的数据完全一样的数据，所以 redis 缓存也要有明确的分页字段

操作 `Reids` 的方法

- **LRANGE**`是 Redis 中用于**获取 List 列表中指定范围的元素**的命令

- **size** 返回 Redis List 中**元素的总个数**
- **rightPushAll**：**追加到尾部**。新查询的数据会排在旧数据后面（符合你的逻辑：查第2页时追加在第1页之后）。
- leftPushAll：**插入到头部**。新数据会插在前面，**整体顺序会颠倒**（常用于“最新评论”置顶的场景）。

空值缓存：如果为空可以给 value 设置为 null，避免查询数据库

```java
if (CollectionUtils.isNotEmpty(pageRecords) && NULL_STR.equals(pageRecords.get(0))) {
    return PageResponseDTO.emptyPage();

```

情景：针对"笔记有评论"vs"笔记无评论"

- 笔记无评论，value 设置为 ["NULL"]，len > 0，但是没有数据，不用查询数据库直接返回[]
- 笔记有评论，则不满足条件

Quest：为什么只缓存前 10 页的数据？

Answer：越往后的评论看的人越少

> *二八定律（帕累托原则）：80% 的读者重点在 20% 的评论上*
