# 分布式锁

## 一、为什么需要锁

### 代码块

```java
public class NoLockDemo {
    private static int counter = 0;
    public static void main(String[] args) throws InterruptedException {
        //启用多个线程,并发的counter++ 10*10000=100000
        Thread[] threads = new Thread[10];
        for (int i = 0; i < 10; i++) {
            threads[i] = new Thread(() -> {
                for (int j = 0; j < 10000; j++) {
                    counter++;
                }
            });
            threads[i].start();
        }
        //主要作用:让所有线程都执行完成先。然后再执行我们的打印结果。
        for (Thread thread : threads) {
            thread.join();
        }
        //期望结果:10*1000=100000
        System.out.println("实际结果:" + counter);
    }
}
```

### 问题分析:

- counter ++不是原子操作，它包含读取、增加、写入三个步骤
- 多线程同时读取到的值可能是相同的,导致增加操作丢失

线程 A ：read counter 的结果为 10，执行 10 + 1 = 11，赋值给 counter = 11

线程 B：并发 read counter 的结果也为 10，执行 10  + 1 = 11，赋值给 counter = 11

很明显多次 ++ 的结果变小了

## 单机锁的弊端

只能锁住"同一进程内"的资源;在分布式环境(多进程/多实例/多机)无法生效,因此需要分布式锁。

# 3.实现锁的本质

锁的本质是在并发环境下对共享资源的访问控制机制。其根本原理可以概括为:
在多个线程或进程需要同时访问某个共享资源时,通过一个全局可见的标记来协调访问顺序,确保在同一时刻只有一个执行单元能够操作该资源,从而保证数据的一致性和操作的原子性。

## 实现锁的技术要素:

- **互斥性**: 同一时刻只有一个客户端能持有锁
- **安全性**: 锁只能由持有者释放,不能被其他客户端释放
- **可用性**: 在持有锁的客户端崩溃时,需要有自动释放机制,避免死锁

# 4.2 Redis实现分布式锁

## 基本原理

想要实现分布式锁,必须要求Redis 有「互斥」的能力,我们可以使用SETNX 命令,这个命令表示SET if Not Exists,即如果key 不存在,才会设置它的值,否则什么也不做。

两个客户端进程可以执行这个命令,达到互斥,就可以实现一个分布式锁。

## 如何避免死锁?

异常情况:在加锁成功后,但是在释放锁之前,程序异常退出,此时锁一直无法释放!那么其他线程将永远无法获取锁。

```bash
setnx lock 1  //step1执行命令加锁
//执行业务逻辑
del lock      //step 2释放锁
```

**解决办法**: 给锁加一个过期时间,一旦过期,锁自动释放

```bash
SETNX lock 1     //加锁
EXPIRE lock 10   //10s后自动过期
```

## 问题:如何保证原子性?

由于setnx 和expire是两个命令,并且分开执行的,那就有可能会出现如下情况:
- SETNX 执行成功,执行EXPIRE 时由于网络问题,执行失败
- SETNX 执行成功,Redis 异常宕机,EXPIRE 没有机会执行
- SETNX 执行成功,客户端异常崩溃,EXPIRE也没有机会执行

总之,这两条命令不能保证是原子操作(一起成功),就有潜在的风险导致过期时间设置失败,依旧发生「死锁」问题。

在Redis 2.6.12之后,Redis 扩展了SET 命令的参数,用这一条命令就可以了:

```bash
SET lock 1 EX 10 NX
```

## 锁被别人释放了怎么办?

我们释放锁是直接执行del lock命令来释放锁,相当于每个客户端(每个线程),只要执行了del lock 就能将这个锁释放掉,因此就会发生释放别人锁的风险,这样的解锁流程,很不「严谨」!如何解决这个问题呢?

**解决办法**是:客户端每次加锁时,需要给锁配一把只有自己知道的"钥匙",解锁时只有拿到了这把钥匙,才能进行解锁。其他线程由于不知道这把锁的钥匙,因此解不了锁。那么,什么东西可以作为"钥匙"?可以是自己的线程id,也可以是uuid。

- **加锁**
```bash
SET lock $uuid EX 20 NX  //加锁
```

- **释放锁**
```bash
if get lock == $uuid:     //判断钥匙准确性
    del lock              //释放锁
```

**问题**: 释放锁的逻辑是,先判断锁的准确性(GET),成功后,执行DEL命令释放锁。因此,释放锁使用的是 GET + DEL两条命令,这时,又会遇到我们前面讲的原子性问题了。

- **使用Lua脚本,保证多命令执行的原子性**

```lua
if redis.call("GET",KEYS[1])==ARGV[1]
then
    return redis.call("DEL",KEYS[1])
else
    return 0
end
```

# 4.2.1 Redis+Lua实现分布式锁

## 代码块

```java
import jakarta.annotation.Resource;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.core.script.DefaultRedisScript;
import org.springframework.stereotype.Component;
import java.util.Collections;
import java.util.UUID;
import java.util.concurrent.TimeUnit;

/**
*基于RedisTemplate 实现的分布式锁
*<p>
*特性:
*-支持可重入(同一线程可多次获取同一个锁)
*-支持自动过期(默认5秒,防止死锁)
*-支持超时等待获取锁
*-使用Lua 脚本保证释放操作的原子性
*
*@Author EMing
*/
@Component
public class RedisDistLock {
    /**
    *锁的默认超时时间:5秒
    */
    private final static int LOCK_TIME = 5 * 1000;
    
    /**
    *分布式锁的Redis key 命名空间
    */
    private final static String RS_DISTLOCK_NS = "rdn:";
    
    /**
    *释放锁的Lua 脚本
    *作用:保证get 和del 操作的原子性
    *逻辑:只有当锁的持有者是当前线程时,才能删除锁
    */
    private final static String RELEASE_LOCK_LUA = 
        "if redis.call('get',KEYS[1])==ARGV[1]then\n" +
        "return redis.call('del',KEYS[1])\n" +
        "else return 0end";
    
    /**
    *保存每个线程独有的ID值
    *用于标识锁的持有者
    */
    private ThreadLocal<String> lockerId = new ThreadLocal<>();
    
    /**
    *当前持有锁的线程
    *用于解决锁的重入问题
    */
    private Thread ownerThread;
    
    /**
    *锁的名称
    *默认为"lock"
    */
    private String lockName = "lock";
    
    /**
    *RedisTemplate 实例
    *使用String 类型的序列化器
    */
    @Resource
    private RedisTemplate<String,String> stringRedisTemplate;
    
    public String getLockName() {
        return lockName;
    }
    
    public void setLockName(String lockName) {
        this.lockName = lockName;
    }
    
    public Thread getOwnerThread() {
        return ownerThread;
    }
    
    public void setOwnerThread(Thread ownerThread) {
        this.ownerThread = ownerThread;
    }
    
    /**
    *尝试获取锁(非阻塞)
    *<p>
    *立即尝试获取锁,不等待。
    *如果获取失败,直接返回false。
    *
    *@return true-获取成功,false-获取失败
    */
    public boolean tryLock() {
        Thread currentThread = Thread.currentThread();
        //如果当前线程已经持有锁,直接返回true(可重入)
        if (ownerThread == currentThread) {
            return true;
        } else if (ownerThread != null) {
            //本进程里有其他线程持有分布式锁
            return false;
        }
        try {
            //生成唯一的锁ID
            String id = UUID.randomUUID().toString();
            //构造Redis key
            String redisKey = RS_DISTLOCK_NS + lockName;
            //线程本地抢锁(避免同一个进程内的多个线程同时操作Redis)
            synchronized (this) {
                //双重检查,确保没有其他线程已经获取了锁
                if (ownerThread == null) {
                    //使用SET NX PX 命令尝试获取锁
                    //NX:只在key 不存在时设置
                    //PX:设置过期时间(毫秒)
                    Boolean result = stringRedisTemplate.opsForValue()
                        .setIfAbsent(redisKey, id, LOCK_TIME, TimeUnit.MILLISECONDS);
                    if (Boolean.TRUE.equals(result)) {
                        //获取锁成功
                        lockerId.set(id);
                        setOwnerThread(currentThread);
                        return true;
                    } else {
                        //获取锁失败
                        return false;
                    }
                } else {
                    return false;
                }
            }
        } catch (Exception e) {
            throw new RuntimeException("分布式锁尝试加锁失败!", e);
        }
    }
    
    /**
    *在指定时间内尝试获取锁(支持超时等待)
    *<p>
    *在超时时间内重复尝试获取锁,如果获取成功则返回true。
    *如果超时时间到期仍未获取到锁,则返回false。
    *
    *@param time 等待时间
    *@param unit 时间单位
    *@return true-获取成功,false-超时未获取到锁
    *@throws InterruptedException 如果等待过程中线程被中断
    */
    public boolean tryLock(long time, TimeUnit unit) throws InterruptedException {
        //计算超时截止时间
        long deadline = System.currentTimeMillis() + unit.toMillis(time);
        //在截止时间前不断尝试获取锁
        while (System.currentTimeMillis() < deadline) {
            //尝试获取锁
            if (tryLock()) {
                return true;
            }
            //未获取到锁,休眠一段时间后重试
            //使用较短的休眠时间以提高响应性
            Thread.sleep(50);
        }
        //超时仍未获取到锁
        return false;
    }
    
    /**
    *释放锁
    *<p>
    *只有锁的持有者才能释放锁,否则抛出异常。
    *使用Lua 脚本确保检查和删除操作的原子性。
    *
    *@throws RuntimeException 如果当前线程不是锁的持有者
    */
    public void unlock() {
        //检查当前线程是否是锁的持有者
        if (ownerThread != Thread.currentThread()) {
            throw new RuntimeException("试图释放无所有权的锁!");
        }
        try {
            //构造Redis key
            String redisKey = RS_DISTLOCK_NS + lockName;
            //创建Lua 脚本对象
            DefaultRedisScript<Long> redisScript = new DefaultRedisScript<>();
            redisScript.setScriptText(RELEASE_LOCK_LUA);
            redisScript.setResultType(Long.class);
            //执行Lua 脚本释放锁
            //KEYS[1]=redisKey
            //ARGV[1]=lockerId
            Long result = stringRedisTemplate.execute(redisScript, 
                Collections.singletonList(redisKey), lockerId.get());
            if (result != null && result != 0L) {
                System.out.println("Redis上的锁已释放!");
            } else {
                System.out.println("Redis上的锁释放失败!");
            }
        } catch (Exception e) {
            throw new RuntimeException("释放锁失败!", e);
        } finally {
            //清理本地状态
            lockerId.remove();
            setOwnerThread(null);
            System.out.println("本地锁所有权已释放!");
        }
    }
}
```

# 4.2.2 Redis "看门狗"分布式锁方案

## 代码块

```java

import jakarta.annotation.Resource;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.core.script.DefaultRedisScript;
import org.springframework.stereotype.Component;
import java.util.Collections;
import java.util.UUID;
import java.util.concurrent.DelayQueue;
import java.util.concurrent.TimeUnit;

/**
*带看门狗线程的分布式锁实现
*<p>
*核心特性:
*-自动续期:看门狗线程在锁快过期前自动续期
*-短过期时间:默认1秒,适合快速释放资源
*-可重入:支持同一线程多次获取同一个锁
*-原子操作:使用Lua 脚本保证操作的原子性
*-自动清理:应用关闭时自动中断看门狗线程
*<p>
*使用场景:
*适用于业务逻辑执行时间不确定或可能超过锁过期时间的场景。
*看门狗会自动续期,避免业务执行过程中锁被提前释放。
*
*@Author green-note
*@Date 2025/01/11
*/

public class RedisDistLockWithDog {
    /**
    *锁的默认过期时间:1秒
    *看门狗会在到期前自动续期
    */

    /**
    *锁过期时间的字符串形式(用于Lua 脚本)
    */
    private final static String LOCK_TIME_STR = String.valueOf(LOCK_TIME);
    
    /**
    *分布式锁的Redis key 命名空间
    */

    /**
    *释放锁的Lua 脚本
    *逻辑:只有锁的持有者才能释放锁
    */

        "if redis.call('get',KEYS[1])==ARGV[1]then\n" +
        "return redis.call('del',KEYS[1])\n" +
        "else return 0end";
    
    /**
    *续期锁的Lua 脚本
    *逻辑:只有锁的持有者才能续期
    */
    private final static String DELAY_LOCK_LUA = 
        "if redis.call('get',KEYS[1])==ARGV[1]then\n" +
        "return redis.call('pexpire',KEYS[1],ARGV[2])\n" +
        "else return 0end";
    
    /**
    *保存每个线程独有的ID 值
    *用于标识锁的持有者
    */
    private ThreadLocal<String> lockerId = new ThreadLocal<>();
    
    /**
    *当前持有锁的线程
    *用于实现锁的可重入
    */
    private Thread ownerThread;
    
    /**
    *锁的名称
    */
    private String lockName = "lock";
    
    /**
    *RedisTemplate 实例
    */
    @Resource
    private RedisTemplate<String,String> stringRedisTemplate;
    
    /**
    *看门狗线程
    *负责在锁快过期前自动续期
    */
    private Thread expireThread;
    
    /**
    *延迟阻塞队列
    *存储需要续期的锁,避免无谓的轮询,减少看门狗线程的轮询次数
    */
    private static DelayQueue<ItemVo<LockItem>> delayDog = new DelayQueue<>();
    
    public String getLockName() {
        return lockName;
    }
    
    public void setLockName(String lockName) {
        this.lockName = lockName;
    }
    
    public Thread getOwnerThread() {
        return ownerThread;
    }
    
    public void setOwnerThread(Thread ownerThread) {
        this.ownerThread = ownerThread;
    }
    
    /**
    *尝试获取锁(非阻塞)
    *
    *@return true-获取成功,false-获取失败
    */
    public boolean tryLock() {
        Thread t = Thread.currentThread();
        //可重入检查:本线程正在持有锁
        if (ownerThread == t) {
            return true;
        } else if (ownerThread != null) {
            //本进程中有别的线程正在持有分布式锁
            return false;
        }
        try {
            //每一个锁的持有人都分配一个唯一的ID
            String id = UUID.randomUUID().toString();
            //构造Redis key
            String redisKey = RS_DISTLOCK_NS + lockName;
            //线程本地抢锁(避免同一个进程内的多个线程同时操作Redis)

                //双重检查
                if (ownerThread == null) {
                    //使用SET NX PX 命令尝试获取锁

                        .setIfAbsent(redisKey, id, LOCK_TIME, TimeUnit.MILLISECONDS);
                    if (Boolean.TRUE.equals(result)) {
                        //获取锁成功
                        lockerId.set(id);
                        setOwnerThread(t);
                        //启动看门狗线程(如果还未启动)
                        if (expireThread == null) {
                            expireThread = new Thread(new ExpireTask(), "WatchDog-ExpireThread");
                            expireThread.setDaemon(true);//设置为守护线程
                            expireThread.start();
                        }
                        //往延迟阻塞队列中加入元素
                        //让看门狗可以在过期之前一点点的时间去做锁的续期

                        System.out.println(Thread.currentThread().getName() + "已获得锁----");
                        return true;
                    } else {
                        System.out.println(Thread.currentThread().getName() + "无法获得锁----");
                        return false;
                    }
                } else {
                    return false;
                }
            }
        } catch (Exception e) {
            throw new RuntimeException("分布式锁尝试加锁失败!", e);
        }
    }

    *在指定时间内尝试获取锁(支持超时等待)
    *
    *@param time 等待时间
    *@param unit 时间单位
    *@return true-获取成功,false-超时未获取到锁
    *@throws InterruptedException 如果等待过程中线程被中断
    */
    public boolean tryLock(long time, TimeUnit unit) throws InterruptedException {
        //计算超时截止时间
        long deadline = System.currentTimeMillis() + unit.toMillis(time);
        //在截止时间前不断尝试获取锁
        while (System.currentTimeMillis() < deadline) {
            if (tryLock()) {
                return true;
            }
            //未获取到锁,休眠一段时间后重试
            Thread.sleep(50);
        }
        //超时仍未获取到锁
        return false;
    }
    
    /**
    *释放锁
    *
    *@throws RuntimeException 如果当前线程不是锁的持有者
    */
    public void unlock() {
        //检查当前线程是否是锁的持有者
        if (ownerThread != Thread.currentThread()) {
            throw new RuntimeException("试图释放无所有权的锁!");
        }
        try {
            //构造Redis key
            String redisKey = RS_DISTLOCK_NS + lockName;
            //创建Lua 脚本对象
            DefaultRedisScript<Long> redisScript = new DefaultRedisScript<>();
            redisScript.setScriptText(RELEASE_LOCK_LUA);
            redisScript.setResultType(Long.class);
            //执行Lua 脚本释放锁
            Long result = stringRedisTemplate.execute(redisScript, 
                Collections.singletonList(redisKey), lockerId.get());
            if (result != null && result != 0L) {
                System.out.println("Redis 上的锁已释放!");
            } else {
                System.out.println("Redis 上的锁释放失败!");
            }
        } catch (Exception e) {
            throw new RuntimeException("释放锁失败!", e);
        } finally {
            //清理本地状态
            lockerId.remove();
            setOwnerThread(null);
        }
    }
    
    /**
    *看门狗任务
    *负责从延迟队列中取出快过期的锁,并进行续期操作
    */
    private class ExpireTask implements Runnable {
        @Override
        public void run() {
            System.out.println("看门狗线程已启动......");
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    //从延迟队列中取出元素(阻塞式)
                    //只有元素快到期了才能take 到(提前100ms)
                    LockItem lockItem = delayDog.take().getData();
                    try {
                        //构造Redis key
                        String redisKey = RS_DISTLOCK_NS + lockItem.getKey();
                        //创建续期的Lua 脚本对象
                        DefaultRedisScript<Long> redisScript = new DefaultRedisScript<>();
                        redisScript.setScriptText(DELAY_LOCK_LUA);
                        redisScript.setResultType(Long.class);
                        //执行Lua 脚本续期
                        //KEYS[1]=redisKey
                        //ARGV[1]=lockItem.getValue()(锁的ID)
                        //ARGV[2]=LOCK_TIME_STR (续期时长)
                        Long result = stringRedisTemplate.execute(redisScript,
                            Collections.singletonList(redisKey), 
                            lockItem.getValue(), LOCK_TIME_STR);
                        if (result != null && result == 0L) {
                            //续期失败,说明锁已经被释放
                            System.out.println("Redis 上的锁已释放,无需续期!");
                        } else {
                            //续期成功,再次将锁加入延迟队列
                            delayDog.add(new ItemVo<>(LOCK_TIME, 
                                new LockItem(lockItem.getKey(), lockItem.getValue())));
                            System.out.println("Redis 上的锁已续期:" + LOCK_TIME + "ms");
                        }
                    } catch (Exception e) {
                        throw new RuntimeException("锁续期失败!", e);
                    }
                } catch (InterruptedException e) {
                    System.out.println("看门狗线程被中断");
                    break;
                }
            }
            System.out.println("看门狗线程准备关闭......");
        }
    }
    
    /**
    *应用关闭时,中断看门狗线程
    */
    @PreDestroy
    public void closeExpireThread() {
        if (null != expireThread) {
            expireThread.interrupt();
        }
    }
}

/**
*类说明:存放到延迟队列的元素,比标准的delay的实现要提前一点时间
*/
public class ItemVo<T> implements Delayed {
    /*到期时刻*/
    private long activeTime;
    /*业务数据,泛型*/
    private T data;
    
    /*传入的数值代表过期的时长,单位毫秒
    *同时提前100毫秒续期,具体的时间可以自己决定*/
    public ItemVo(long expirationTime, T data) {
        super();
        this.activeTime = expirationTime + System.currentTimeMillis() - 100;
        this.data = data;
    }
    
    public long getActiveTime() {
        return activeTime;
    }
    
    public T getData() {
        return data;
    }
    
    /**
    *返回元素到激活时刻的剩余时长
    */
    public long getDelay(TimeUnit unit) {
        long d = unit.convert(this.activeTime - System.currentTimeMillis(), unit);
        return d;
    }
    
    /**按剩余时长排序*/
    public int compareTo(Delayed o) {
        long d = (getDelay(TimeUnit.MILLISECONDS) - o.getDelay(TimeUnit.MILLISECONDS));
        if (d == 0) {
            return 0;
        } else {
            if (d < 0) {
                return -1;
            } else {
                return 1;
            }
        }
    }
}
```

# 4.2.4 红锁

单实例--单点故障
- 主从模式+哨兵
- 使用集群

解决redis单点故障问题,RedLock 使用了多个独立的Redis 实例(通常建议是奇数个,比如5个),共同协作来提供更强健的分布式锁服务。

## Redlock实现整体流程

1. 客户端获取锁【当前时间戳T1】
2. 客户端一次向这5台redis实例(非集群)发起加锁请求
3. 如果>=3个(大多数)成功,当前时间戳T2-T1<锁的过期时间则加锁成功
4. 加锁成功,执行业务逻辑,去操作共享资源
5. 加锁失败/释放锁,向5台Redis发起释放锁请求

## RedLock的NPC问题

- **N**: Network delay 网络延迟
- **P**: Process pause 进程暂停(GC)
- **C**: Clock Drift 时钟漂移

# 4.3 zookeeper分布式锁实现原理

1. **互斥性**: 文件系统决定它天然互斥
2. **容错性**: 使用zk临时节点(会话断开时,临时节点会被删除掉)
3. **安全性**: 只有持有当前锁的线程才能释放锁(避免锁被其他线程释放)
4. **锁释放了,如何告知到其他线程?**
   - 应用程序使用轮训请求的方式,查看临时节点是否存在。若存在,则说明锁还没被释放,若不存在则可以直接争抢锁。(会对zk造成压力,延迟问题)
   - watch机制
   - 序号节点

//ZookeeperLockDemo.java 完整代码
import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.apache.curator.framework.recipes.locks.InterProcessMutex;
import java.util.concurrent.TimeUnit;

/**
*使用Curator 的InterProcessMutex 实现分布式锁。
*ZK 基础:
*-临时节点(Ephemeral):会话断开后自动删除,适合表示持有锁的状态;
*-临时顺序节点(EphemeralSequential):用于公平排队;
*-Watch 机制:监听节点变化以唤醒等待方。
*/
public class ZookeeperLockDemo {
    private static CuratorFramework client() {
        //演示:本地ZK 默认127.0.0.1:2181
        return CuratorFrameworkFactory.builder()
            .connectString("127.0.0.1:2181")
            .retryPolicy(new ExponentialBackoffRetry(1000, 3))
            .build();
    }
    
    public static void main(String[] args) throws Exception {
        CuratorFramework client = client();
        client.start();
        String lockPath = "/demo/locks/order-123";//对应一个业务资源
        InterProcessMutex lock = new InterProcessMutex(client, lockPath);
        //tryLock 带超时,避免永久阻塞
        if (lock.acquire(3, TimeUnit.SECONDS)) {
            try {
                System.out.println("获取到ZK 锁,执行业务...");
                Thread.sleep(1500);
            } finally {
                lock.release();
                System.out.println("释放ZK 锁完成");
            }
        } else {
            System.out.println("未获取到ZK 锁,稍后重试");
        }
        client.close();
    }
}
